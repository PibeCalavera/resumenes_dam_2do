1. Conceptos generales de multimedia

1.1. Tipos de contenido multimedia

  • Texto / Hipertexto

    • Definición: información lineal (texto) o enriquecida con enlaces, referencias y metadatos (hipertexto).

    • Uso: presentación de información acompañante, etiquetas de medios, subtítulos, menús.

    • Ejemplo: títulos de canciones, descripciones de vídeo, menús interactivos.

  • Imágenes

    • Definición: representaciones gráficas fijas de información visual.

    • Formatos comunes:

      • PNG: sin pérdida, soporta transparencia; recomendable para iconos y gráficos.

      • JPEG: compresión con pérdida, tamaño reducido; recomendable para fotografías.

      • WebP: compresión eficiente, soporta transparencia y animación; útil para apps modernas Android.

    • Uso: fondos, iconos, ilustraciones, imágenes dentro de apps multimedia.

  • Vídeos

    • Definición: secuencia de imágenes en movimiento con posible audio.

    • Contenedores y códecs comunes:

      • MP4 / 3GPP: H.264 para vídeo, AAC para audio; alto soporte Android.

      • MKV / WebM: contenedores flexibles, VP8/VP9 o H.264; útiles para streaming o apps con multimedia avanzada.

    • Uso: tutoriales, clips, animaciones complejas.

    • Notas: el contenedor define cómo se empaquetan pistas; los códecs determinan la compresión y calidad.

  • Audio / Música

    • Definición: grabaciones de sonidos naturales o generados por ordenador.

    • Formatos comunes:

      • AAC: buena calidad, eficiente, estándar Android/iOS.

      • AMR_NB / AMR_WB: optimizados para voz; tamaño reducido.

      • MP3: ampliamente soportado, compresión con pérdida.

      • FLAC / Opus: sin pérdida o alta eficiencia, recomendable para apps de música de alta calidad.

      • MIDI: secuencias de notas y controladores, no audio real; útil para sintetizadores.

    • Uso: música de fondo, efectos de sonido, voz.

    • Consideración: elegir formato según calidad deseada, tamaño y compatibilidad.

  • Animaciones

    • Definición: presentaciones gráficas que crean ilusión de movimiento.

    • Tipos:

      • Basadas en vista: movimientos simples de objetos ( res/anim ).

      • Basadas en propiedades: interpolación de atributos ( translationX, alpha, scale ) con ObjectAnimator o ViewPropertyAnimator ( res/animator ).

    • Uso: transiciones de UI, efectos visuales, animación de objetos en apps interactivas.

1.2. Fuentes de datos multimedia

  • Recursos incluidos en la app

    • Ubicación: res/raw/, res/animator/, etc.

    • Características: siempre disponibles, no dependen de almacenamiento externo ni red.

    • Uso recomendado: elementos que se usan con frecuencia, como sonidos cortos o clips incluidos en la app.

  • Ficheros locales

    • Ubicación: almacenamiento interno o externo ( getFilesDir(), getExternalFilesDir() ).

    • Características: permite archivos grandes; acceso como ficheros normales.

    • Uso recomendado: vídeos descargados, grabaciones de usuario, contenido de gran tamaño.

  • Recursos de red / Streaming

    • Características: se accede vía internet (HTTP, RTSP), depende de ancho de banda.

    • Uso: reproducción en tiempo real sin descargar el archivo completo; streaming de audio o vídeo.

    • Consideraciones: controlar buffering, manejo de errores de red, adaptar calidad según conexión.

  • Nota: no todas las fuentes posibles se limitan a estas; por ejemplo, APIs de terceros pueden proporcionar streams en vivo o contenido dinámico.

Este bloque cubre los conceptos generales y la base de todo el contenido multimedia: tipos de medios y fuentes de datos.

  • A partir de aquí se entenderán mejor las decisiones sobre clases, métodos y formatos en reproducción y grabación.

  • Hemos incluido ejemplos y criterios de uso, no solo definiciones.

2. Reproducción multimedia

2.1. Clases de reproducción

  • MediaPlayer

    • Definición: Clase principal para reproducir audio y vídeo desde distintas fuentes en Android.

    • Fuentes soportadas: tarjeta SD, memoria interna, streaming de red (http/rtsp).

    • Estados: Idle → Initialized → Prepared → Started → Paused → Stopped → End

    • Métodos principales:

      • setDataSource(): define la fuente del medio.

      • prepare() / prepareAsync(): prepara la reproducción; Async evita bloqueo de UI.

      • start(): inicia la reproducción.

      • pause(): pausa la reproducción, mantiene la posición actual.

      • stop(): detiene la reproducción; requiere re-preparación para reiniciar.

      • seekTo(int msec): mueve la posición a un instante específico; solo válido en estado Prepared o Started.

      • release(): libera todos los recursos del reproductor, evitando fugas de memoria.

    • Listeners importantes:

      • OnPreparedListener: se activa cuando el medio está listo para reproducirse.

      • OnCompletionListener: se activa al finalizar la reproducción; útil para reiniciar audio o ejecutar acciones finales.

      • OnBufferingUpdateListener: indica progreso del buffer en streaming.

      • OnErrorListener: captura errores durante la reproducción.

  • SoundPool

    • Definición: Reproducción de sonidos cortos con baja latencia.

    • Uso recomendado: efectos de sonido, notificaciones, alertas cortas.

  • AudioTrack

    • Definición: Reproducción de audio desde buffers PCM; permite streaming de audio en tiempo real.

    • Uso recomendado: audio generado dinámicamente o streaming de datos binarios.

  • VideoView

    • Definición: Vista que simplifica la reproducción de vídeo.

    • Control: se puede enlazar a un MediaController para obtener play, pause, stop y seek.

    • Método clave: setMediaController(MediaController controller).

  • ExoPlayer (ampliación sugerida)

    • Definición: Reproductor avanzado de código abierto, soporta HLS, DASH, MP4, streaming adaptativo.

    • Ventaja: manejo eficiente de buffering y formatos modernos; recomendable para apps de streaming profesional.

2.2. Control de reproducción

  • MediaController

    • Definición: Interfaz gráfica que proporciona botones de control estándar (play, pause, stop, seek).

    • Uso: asociado a VideoView o MediaPlayer para facilitar interacción con el usuario.

  • seekTo()

    • Función: mueve la posición de reproducción a un instante específico.

    • Requisitos: solo puede invocarse en estados Prepared o Started.

    • Uso práctico: reiniciar audio, saltar a un punto específico en un vídeo.

  • OnCompletionListener

    • Función: se activa al finalizar la reproducción de un medio.

    • Uso práctico: reiniciar audio, cambiar al siguiente elemento de una lista de reproducción.

2.3. Streaming

  • Definición: Reproducción de audio o vídeo en tiempo real, sin descargar el archivo completo.

  • Características:

    • Requiere conectividad de red.

    • Calidad puede adaptarse según ancho de banda.

    • Control de buffering mediante OnBufferingUpdateListener.

  • Uso recomendado: radio online, servicios de vídeo bajo demanda, streaming de eventos en vivo.

3. Grabación multimedia

3.1. Clase principal

  • MediaRecorder

    • Definición: Clase que permite capturar audio y vídeo en Android y almacenarlo en ficheros.

    • Uso recomendado: grabación de audio para notas, música, voz del usuario, o captura de vídeo con la cámara.

    • Ventajas: control de fuente, formato, codificador, duración y tamaño de archivo; integración nativa con Android.

3.2. Métodos principales

  • setAudioSource(int audio_source)

    • Función: define la fuente de audio.

    • Valores típicos: MIC (micrófono), CAMCORDER, DEFAULT, VOICE_CALL.

  • setVideoSource(int video_source)

    • Función: define la fuente de vídeo.

    • Valores típicos: CAMERA, SURFACE.

  • setOutputFile(String file)

    • Función: define el fichero donde se guardará la grabación.

  • setOutputFormat(int format)

    • Función: define el contenedor del archivo de salida.

    • Valores típicos: MPEG_4, THREE_GPP, AMR_NB, AMR_WB, RAW_AMR.

  • setAudioEncoder(int encoder)

    • Función: define la codificación de audio.

    • Valores típicos: AAC, AMR_NB, AMR_WB, DEFAULT.

  • setVideoEncoder(int encoder)

    • Función: define la codificación de vídeo.

    • Valores típicos: H264, H263, DEFAULT.

  • setAudioChannels(int channels)

    • Función: define el número de canales de audio.

    • Valores típicos: 1 = mono, 2 = estéreo.

  • setAudioEncodingBitRate(int bitRate)

    • Función: define los bits por segundo de la codificación.

  • setAudioSamplingRate(int samplingRate)

    • Función: define la frecuencia de muestreo de audio.

  • setMaxDuration(int max_duration_ms)

    • Función: define duración máxima de grabación; se detiene automáticamente al alcanzarla.

  • setMaxFileSize(long max_filesize_bytes)

    • Función: define tamaño máximo de archivo; se detiene automáticamente al alcanzarlo.

  • prepare()

    • Función: prepara el objeto para comenzar la captura de datos; debe llamarse después de configurar fuentes, formato y codificadores.

  • start()

    • Función: inicia la captura de audio y/o vídeo.

  • stop()

    • Función: finaliza la grabación.

  • reset()

    • Función: reinicia el objeto como si fuera nuevo; obliga a reconfigurar.

  • release()

    • Función: libera todos los recursos inmediatamente, evitando fugas de memoria.

3.3. Permisos

  • RECORD_AUDIO → permite grabar audio desde el micrófono.

  • WRITE_EXTERNAL_STORAGE → permite guardar la grabación en almacenamiento externo.

  • READ_EXTERNAL_STORAGE → permite acceder a archivos existentes en almacenamiento externo.

  • Nota: desde API 23 (Android 6 – Marshmallow) los permisos peligrosos deben solicitarse en tiempo de ejecución, no solo en el AndroidManifest.xml.

3.4. Manipulación de contenedores

  • MediaExtractor + MediaMuxer

    • Definición: permiten extraer pistas de un contenedor y reempaquetarlas en otro formato.

    • Uso: reempaquetar audio y vídeo, cambiar contenedor sin recodificar, separar pistas para edición.

3.5. Consideraciones y buenas prácticas

  • Liberar recursos: llamar a release() siempre que no se necesite más el objeto.

  • Reset antes de reconfigurar: permite reutilizar un objeto MediaRecorder.

  • Seleccionar codificador y formato según uso:

    • AAC + MPEG_4 → buena compatibilidad y calidad para audio/video general.

    • AMR → optimizado para voz, bajo tamaño.

    • H264 + MPEG_4 → vídeo de calidad estándar y buena compatibilidad.

  • Control de duración y tamaño: evita grabaciones demasiado largas o ficheros gigantes.

4. Ciclo de vida de Activities y eventos

4.1. Métodos principales de ciclo de vida

  • onCreate(Bundle savedInstanceState)

    • Función: se activa al crear la Activity por primera vez.

    • Uso típico: inicializar UI, inflar layouts, inicializar variables y objetos multimedia ( MediaPlayer, MediaRecorder ).

  • onStart()

    • Función: se activa cuando la Activity se hace visible para el usuario.

    • Uso típico: preparar recursos que deben estar disponibles mientras la Activity es visible.

  • onResume()

    • Función: se activa cuando la Activity pasa a primer plano y puede interactuar con el usuario.

    • Uso típico: reanudar reproducción de audio o vídeo, reiniciar animaciones, reactivar sensores.

  • onPause()

    • Función: se activa cuando el usuario está dejando la Activity, que puede quedar parcialmente visible.

    • Uso típico: pausar reproducción multimedia, guardar posición de MediaPlayer, detener animaciones temporales.

  • onStop()

    • Función: se activa cuando la Activity ya no es visible para el usuario.

    • Uso típico: liberar recursos que consumen memoria o CPU (como MediaPlayer o MediaRecorder ).

  • onDestroy()

    • Función: se activa antes de que la Activity sea destruida.

    • Uso típico: liberar completamente objetos multimedia y otros recursos críticos.

4.2. Interacción con Multimedia

  • MediaPlayer y ciclo de vida:

    • Guardar posición actual en onPause() si se desea continuar después.

    • Llamar a release() en onStop() o onDestroy() para liberar memoria.

    • Eventos importantes:

      • OnPreparedListener → medio listo para reproducirse.

      • OnCompletionListener → fin de reproducción; útil para reiniciar audio o cambiar pista.

      • OnBufferingUpdateListener → progreso del buffer en streaming.

  • MediaRecorder y ciclo de vida:

    • Llamar a release() en onStop() o onDestroy().

    • reset() permite reutilizar el objeto antes de reconfigurar.

  • Cambio de orientación:

    • Por defecto, MediaPlayer pierde posición al cambiar orientación.

    • Guardar/restaurar posición permite continuar reproducción sin interrupciones.

4.3. Buenas prácticas

  • Liberar recursos pesados en onStop() o onDestroy().

  • Guardar estado de reproducción en onPause() para restaurarlo en onResume().

  • Listeners: asegurarse de registrarlos/desregistrarlos para evitar fugas de memoria.

  • Permisos: verificar que los permisos de almacenamiento y micrófono están disponibles antes de usar objetos multimedia.

5. Animaciones

5.1. Tipos de animaciones en Android

  • Animaciones basadas en vistas

    • Definición: modifican la apariencia de un objeto ( View ) usando transformaciones predefinidas (traslación, rotación, escala, alfa).

    • Recursos XML: se guardan en res/anim/.

    • Clases relevantes:

      • Animation: clase base para animaciones tradicionales.

      • AnimationUtils: carga animaciones desde XML.

    • Uso recomendado: movimientos simples de UI, transiciones de botones, fades, rotaciones básicas.

  • Animaciones basadas en propiedades

    • Definición: interpolan cualquier propiedad de un objeto (posición, opacidad, tamaño, color).

    • Recursos XML: se guardan en res/animator/.

    • Clases relevantes:

      • ObjectAnimator: anima una propiedad específica de un objeto, ej. translationX, alpha.

      • ValueAnimator: genera valores numéricos interpolados; puede aplicarse a cualquier atributo mediante callbacks.

      • AnimatorSet: combina varias animaciones para reproducirse en secuencia o simultáneamente.

      • ViewPropertyAnimator: API simplificada para animar varias propiedades de un View directamente.

    • Uso recomendado: animaciones complejas de UI, transiciones fluidas, interacciones dinámicas.

5.2. Animaciones entre Activities

  • ActivityOptions

    • Definición: objeto que configura animaciones de transición entre Activities.

    • Uso típico: efectos de entrada/salida al iniciar o cerrar Activities.

  • Transition

    • Definición: clase que define animaciones de cambio de estado de la UI, usada junto con ActivityOptions o Scene.

    • Uso recomendado: transiciones avanzadas entre layouts o Activities, animaciones de elementos compartidos.

  • Buenas prácticas:

    • Usar animaciones por propiedades cuando se necesite rendimiento y GPU acelerada.

    • Evitar animaciones pesadas en bucles frecuentes o listas largas para no afectar la UI.

6. Formatos y codificadores

6.1. Formatos de audio

  • AAC (Advanced Audio Coding)

    • Definición: compresión con pérdida, alta eficiencia y buena calidad.

    • Uso recomendado: música, audio de vídeo, streaming; estándar Android/iOS.

    • Clases relacionadas: MediaRecorder.AudioEncoder.AAC.

  • AMR (Adaptive Multi-Rate)

    • Definición: optimizado para voz; bajo tamaño de archivo.

    • Variantes: AMR_NB (narrowband) y AMR_WB (wideband).

    • Uso recomendado: grabaciones de voz, llamadas VoIP.

  • MP3

    • Definición: compresión con pérdida ampliamente soportada.

    • Uso recomendado: música general, efectos de sonido.

  • FLAC

    • Definición: compresión sin pérdida; conserva calidad total.

    • Uso recomendado: música de alta fidelidad.

  • MIDI

    • Definición: secuencias de notas y controladores, no audio real.

    • Uso recomendado: sintetizadores, instrumentos virtuales.

6.2. Formatos de vídeo

  • MP4 / 3GPP

    • Definición: contenedores que empaquetan vídeo y audio; comúnmente H.264 + AAC.

    • Uso recomendado: reproducción general en Android, streaming, almacenamiento local.

  • MKV / WebM

    • Definición: contenedores flexibles; soportan VP8/VP9 y H.264.

    • Uso recomendado: streaming avanzado, vídeos de alta resolución, subtítulos integrados.

  • AVI

    • Definición: contenedor clásico; soporte limitado en Android nativo.

    • Uso recomendado: conversión o compatibilidad legacy.

6.3. Codificadores (Encoders)

  • Audio

    • MediaRecorder.AudioEncoder.AAC → audio general de alta calidad.

    • MediaRecorder.AudioEncoder.AMR_NB / AMR_WB → voz, bajo tamaño.

    • MediaRecorder.AudioEncoder.DEFAULT → selecciona automáticamente el codec por defecto del dispositivo.

  • Vídeo

    • MediaRecorder.VideoEncoder.H264 → estándar moderno, alta compresión y calidad.

    • MediaRecorder.VideoEncoder.H263 → compatibilidad con dispositivos antiguos.

    • MediaRecorder.VideoEncoder.DEFAULT → codec predeterminado del dispositivo.

  • Consideraciones:

    • Elegir codificador según compatibilidad, calidad, tamaño de archivo y tipo de contenido.

    • Evitar formatos o codificadores no soportados por el dispositivo destino para garantizar reproducción.

6.4. Contenedores y relación con codificadores

  • Contenedor: archivo que empaqueta pistas de audio y vídeo, metadatos, subtítulos y capítulos.

  • Ejemplos: MP4, 3GPP, MKV, WebM, AVI.

  • Codificador vs contenedor:

    • El codificador determina cómo se comprime el contenido.

    • El contenedor determina cómo se empaquetan las pistas y qué reproductores pueden abrirlo.

  • Práctica: combinar AAC + MP4, H264 + MP4, VP9 + WebM según el uso esperado.

7. Flujos de audio y emisión en tiempo real

7.1. Clases principales para reproducción de audio

  • AudioTrack

    • Definición: permite reproducir audio a nivel de buffer PCM; útil para streaming o audio generado dinámicamente.

    • Uso recomendado: reproducción de audio en tiempo real, sonidos generados por la app, manipulación avanzada de audio.

    • Métodos clave:

      • write(byte[] buffer, int offset, int size) → envía datos de audio al buffer.

      • play() / pause() / stop() → controla la reproducción.

      • flush() → limpia el buffer de audio.

      • release() → libera recursos.

  • AudioManager

    • Definición: gestiona el audio del dispositivo, niveles de volumen, rutas de salida (altavoz, auriculares, Bluetooth).

    • Uso recomendado: configurar volumen, cambiar salida de audio, detectar cambios en hardware de audio.

  • SoundPool (ya mencionado en reproducción)

    • Función adicional: permite cargar varios clips cortos y reproducirlos simultáneamente con baja latencia.

    • Uso recomendado: efectos de sonido en juegos o notificaciones interactivas.

7.2. Emisión de audio y streaming

  • AudioGroup (API obsoleta)

    • Definición: clase que agrupaba flujos de audio para comunicaciones en tiempo real.

    • Uso histórico: voz en conferencias, llamadas VoIP.

    • Nota: hoy se usan APIs más modernas (WebRTC, MediaCodec, AudioRecord/AudioTrack).

  • RTP (Real-time Transport Protocol)

    • Definición: protocolo de transmisión de audio y vídeo en tiempo real.

    • Uso recomendado: videoconferencias, streaming en directo.

    • Relación con Android: se implementa mediante bibliotecas externas o frameworks especializados (no nativo en MediaPlayer/MediaRecorder).

  • AudioStream / AudioRecord

    • Definición: permite capturar audio desde el micrófono en tiempo real para enviar a otro sistema o procesar en la app.

    • Métodos clave de AudioRecord:

      • startRecording() / stop() → control de captura.

      • read() → obtiene datos PCM del micrófono.

  • Flujo de audio general:

    • Capturar audio con AudioRecord o MediaRecorder.

    • Procesar o comprimir con codificador adecuado (AAC, AMR, PCM).

    • Enviar por red usando RTP o socket.

    • Reproducir en destino con AudioTrack.

7.3. Consideraciones y buenas prácticas

  • Usar buffers adecuados para evitar latencia o glitches.

  • Liberar recursos de AudioTrack y AudioRecord tras su uso.

  • Seleccionar codificador y formato según ancho de banda y compatibilidad del receptor.

  • Para streaming en Android moderno, preferir MediaCodec + AudioTrack o WebRTC frente a AudioGroup obsoleto.

8. Control y monitoreo del buffer y eventos multimedia

8.1. Eventos de MediaPlayer

  • OnPreparedListener

    • Definición: se activa cuando el medio está listo para reproducirse después de llamar a prepare() o prepareAsync().

    • Uso recomendado: iniciar la reproducción automáticamente, habilitar botones de control, preparar UI.

  • OnCompletionListener

    • Definición: se activa al finalizar la reproducción de un medio.

    • Uso recomendado: reiniciar audio, avanzar al siguiente elemento en una lista, liberar recursos o cambiar estado de la UI.

  • OnBufferingUpdateListener

    • Definición: notifica el progreso del buffering cuando se reproduce contenido en streaming.

    • Uso recomendado: mostrar barra de progreso, evitar pausas, optimizar UX en streaming en vivo.

  • OnErrorListener

    • Definición: captura errores durante la reproducción.

    • Uso recomendado: manejar errores de fuente, códec, o red, y ofrecer feedback al usuario.

  • OnSeekCompleteListener

    • Definición: se activa al completar un seekTo() en MediaPlayer.

    • Uso recomendado: reanudar UI o reproducir el medio en la posición deseada.

8.2. Logs y depuración

  • Log de ejecución

    • Definición: registro de eventos de la aplicación, útil para depuración.

    • Uso recomendado: seguimiento de errores, estado de reproducción, posición de medios, inicio y fin de grabaciones.

  • LOG_TAG

    • Definición: constante String usada para identificar las entradas de log.

    • Uso recomendado: filtrar registros específicos de una clase o funcionalidad (ej. audio, vídeo).

8.3. Buenas prácticas de control

  • Liberación de recursos multimedia

    • Momento recomendado: onStop() o onDestroy() de la Activity.

    • Métodos clave:

      • MediaPlayer.release() → libera todos los recursos de forma inmediata.

      • MediaRecorder.release() → similar para objetos de grabación.

  • Permisos y seguridad

    • Verificar permisos en tiempo de ejecución para audio y almacenamiento antes de iniciar reproducción o grabación.

  • Control de flujo de streaming

    • Usar OnBufferingUpdateListener para ajustar reproducción y UX.

    • Monitorizar posición con seekTo() y OnSeekCompleteListener.

    • Manejar errores y eventos inesperados con OnErrorListener.

  • Integración con ciclo de vida de Activities

    • Pausar reproducción en onPause().

    • Guardar estado de reproducción (posición) y restaurar en onResume().

    • Liberar recursos pesados en onStop() o onDestroy().

Resumen rápido – Unidad 03 Multimedia y Dispositivos Móviles

1. Fuentes de datos multimedia

  • Recursos de la app → siempre accesibles; recomendado para uso frecuente.

  • Ficheros locales → interno o externo; usar para archivos grandes.

  • Recursos de red → streaming; depende de conectividad.

2. Reproducción multimedia

  • Clases principales:

    • MediaPlayer → audio/vídeo; estados: Idle → Initialized → Prepared → Started → Paused → Stopped → End

    • VideoView → simplifica reproducción de vídeo; usar con MediaController.

    • SoundPool → efectos cortos; baja latencia.

    • AudioTrack → audio PCM; streaming/datos generados.

  • Control:

    • start(), pause(), stop(), seekTo(), release().

    • Listeners: OnPreparedListener, OnCompletionListener, OnBufferingUpdateListener, OnErrorListener.

  • Streaming → reproducción en tiempo real; usar OnBufferingUpdateListener para barra de progreso.

3. Grabación multimedia

  • Clase principal: MediaRecorder.

  • Métodos clave:

    • setAudioSource(MIC), setVideoSource(CAMERA)

    • setOutputFile(), setOutputFormat(), setAudioEncoder(), setVideoEncoder()

    • prepare(), start(), stop(), reset(), release()

  • Permisos: RECORD_AUDIO, WRITE_EXTERNAL_STORAGE, READ_EXTERNAL_STORAGE (runtime desde API 23).

  • Reempaquetado: MediaExtractor + MediaMuxer.

4. Ciclo de vida de Activities

  • onCreate() → inicialización UI y objetos multimedia.

  • onStart() → Activity visible.

  • onResume() → Activity en primer plano; iniciar/reanudar reproducción.

  • onPause() → pausar reproducción, guardar estado.

  • onStop() → Activity no visible; liberar recursos pesados.

  • onDestroy() → liberar completamente recursos.

5. Animaciones

  • Basadas en vistas → Animation, AnimationUtils; XML en res/anim/.

  • Por propiedades → ObjectAnimator, ValueAnimator, AnimatorSet, ViewPropertyAnimator; XML en res/animator/.

  • Entre Activities → ActivityOptions, Transition.

6. Formatos y codificadores

  • Audio: AAC (general), AMR_NB/WB (voz), MP3, FLAC, MIDI.

  • Vídeo: MP4/3GPP (H264 + AAC), MKV/WebM (VP8/VP9), AVI (legacy).

  • Codificadores: MediaRecorder.AudioEncoder y MediaRecorder.VideoEncoder.

  • Recomendación: elegir codificador según compatibilidad, calidad y tamaño.

7. Flujos de audio y streaming

  • Captura: AudioRecord, MediaRecorder.

  • Reproducción: AudioTrack, SoundPool.

  • Gestión audio: AudioManager.

  • Streaming en tiempo real: RTP, sockets, WebRTC; buffers correctos para evitar latencia.

8. Control y monitoreo

  • Eventos MediaPlayer:

    • OnPreparedListener, OnCompletionListener, OnBufferingUpdateListener, OnSeekCompleteListener, OnErrorListener.

  • Logs: Log.d(LOG_TAG, mensaje).

  • Liberación de recursos: release() en onStop() / onDestroy().

  • Guardado/restauración: posición de reproducción en onPause() / onResume().
