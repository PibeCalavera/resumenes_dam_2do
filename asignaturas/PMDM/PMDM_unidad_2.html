1. Estructura y organización de proyectos Android

Un proyecto Android sigue una estructura jerárquica definida para mantener el código organizado y facilitar la compilación y ejecución de la aplicación. Los elementos clave son:

1.1. Carpetas principales

    • /src/main/java Contiene el código fuente de la aplicación (Java o Kotlin). La convención para nombrar los paquetes es usar el nombre de dominio de la empresa en minúsculas invertido, seguido del nombre de la app o módulo, por ejemplo:

      com.ejemplo.miapp

      Esto evita conflictos de nombres entre aplicaciones diferentes.

    • /res Contiene los recursos de la aplicación: layouts ( layout ), cadenas de texto ( values/strings.xml ), imágenes ( drawable ), estilos ( styles.xml ), colores, animaciones, etc.

1.2. Archivos importantes

    • R.java Es un archivo autogenerado que asigna un identificador único ( id ) a cada recurso. Por ejemplo:

      R.id.botonEnviar
R.layout.activity_main

      Se utiliza para acceder a los recursos desde el código.

    • AndroidManifest.xml Archivo de configuración principal que declara:

      • Actividades ( Activity )

      • Servicios ( Service )

      • Receptores ( BroadcastReceiver )

      • Permisos requeridos ( <uses-permission> )

      • Metadata de la aplicación (nombre, icono, tema, versión).

1.3. Gestión de dependencias y compilación

    • Gradle es el sistema de compilación de Android Studio.

      • build.gradle a nivel de proyecto → declara dependencias globales, plugins y versiones.

      • build.gradle a nivel de módulo → dependencias específicas de la aplicación o módulo.

1.4. APK y App Bundle

    • APK (Android Package) Archivo ejecutable de Android que contiene código compilado, recursos y manifest, listo para instalarse en un dispositivo.

    • App Bundle (AAB) Contiene todos los recursos y código compilado, pero no se instala directamente. Google Play genera APK optimizados por dispositivo desde el App Bundle.

    • Firma digital obligatoria Todos los APK o App Bundles deben estar firmados digitalmente para ser instalados.

    • Generación en Android Studio Menú:

      Build → Build Bundle / APK → Build APK / Build App Bundle

2. Actividades y UI

En Android, la interfaz de usuario (UI) y la interacción con el usuario se gestionan principalmente mediante Activities y sus componentes visuales. Los elementos clave son:

2.1. Activity

    • Es la unidad principal de interacción con el usuario. Cada pantalla de la app suele corresponder a una Activity.

    • Para cargar la interfaz visual definida en XML dentro de una Activity, se utiliza:

      setContentView(R.layout.nombre_layout);

      Esto enlaza el layout con la Activity, mostrando los controles definidos en el archivo XML.

2.2. WebView

    • Componente que permite incrustar contenido web dentro de la interfaz de la app.

    • Útil para mostrar páginas HTML, mapas, o contenido dinámico desde la web sin abrir un navegador externo.

    • Se utiliza así:

      WebView webView = findViewById(R.id.miWebView);
webView.loadUrl("https://www.ejemplo.com");

2.3. Toast

    • Permite mostrar mensajes cortos y no interactivos al usuario.

    • La duración es breve ( Toast.LENGTH_SHORT ) o larga ( Toast.LENGTH_LONG ).

    • Ejemplo de uso:

      Toast.makeText(this, "Mensaje rápido", Toast.LENGTH_SHORT).show();

    • Ideal para notificaciones rápidas o confirmaciones sin interrumpir la interacción con la app.

2.4. Interacción con componentes

    • Los elementos visuales (botones, campos de texto, listas) se referencian desde el código mediante sus IDs definidos en XML ( R.id.nombreElemento ).

    • Se pueden registrar listeners para manejar eventos como clics o gestos.

3. Manejo de recursos y almacenamiento

Android proporciona mecanismos para gestionar recursos de la aplicación y almacenar datos de manera persistente, ya sea en almacenamiento interno, externo o bases de datos locales.

3.1. Recursos de la aplicación

  • Se almacenan en la carpeta /res y se categorizan por tipo:

    • Layouts → XML que define la interfaz ( res/layout/ ).

    • Strings → textos reutilizables ( res/values/strings.xml ).

    • Drawables → imágenes y gráficos ( res/drawable/ ).

    • Colores, estilos y temas → ( res/values/colors.xml, styles.xml ).

  • Acceso a recursos: todos los recursos tienen un identificador en R.java. Ejemplos:

    String titulo = getString(R.string.app_name);
Button boton = findViewById(R.id.miBoton);

3.2. Almacenamiento de datos

Android permite almacenar datos en tres niveles principales:

a) Archivos

  • Lectura/escritura de archivos mediante clases estándar de Java:

    FileInputStream fis = openFileInput("miArchivo.txt");
int byteLeido = fis.read();
fis.close();

  • Para escribir en almacenamiento externo, es necesario el permiso:

    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />

b) Bases de datos SQLite

  • Android integra SQLite como base de datos local ligera.

  • Acceso a la base de datos:

    • Solo lectura:

      SQLiteDatabase db = dbHelper.getReadableDatabase();

    • Escritura:

      SQLiteDatabase db = dbHelper.getWritableDatabase();

  • Consultas parametrizadas:

    Cursor cursor = db.rawQuery("SELECT * FROM tabla WHERE columna=?", new String[]{valor});

  • Ejecución de sentencias SQL sin retorno:

    db.execSQL("INSERT INTO tabla(columna) VALUES('dato')");

c) Bundles y almacenamiento en memoria

  • Para pasar datos entre Activities se usan Bundles, especialmente en Intents:

    Intent intent = new Intent(this, OtraActivity.class);
intent.putExtra("clave", valor);
startActivity(intent);

    • Los datos quedan almacenados temporalmente en memoria durante la transición.

4. Permisos y seguridad

Android implementa un modelo de seguridad basado en permisos, que controlan el acceso de la aplicación a recursos sensibles del dispositivo o del usuario.

4.1. Declaración de permisos

  • Todos los permisos que la aplicación necesita se declaran en el AndroidManifest.xml:

    <uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>

  • Esto permite que el sistema sepa qué recursos requiere la aplicación antes de instalarla.

4.2. Tipos de permisos según funcionalidad

  • Acceso a internet:

    <uses-permission android:name="android.permission.INTERNET"/>

  • Ubicación:

    • Precisa: ACCESS_FINE_LOCATION → acceso GPS exacto.

    • Aproximada: ACCESS_COARSE_LOCATION → basada en red o Wi-Fi.

  • Almacenamiento externo:

    • Escritura: WRITE_EXTERNAL_STORAGE

    • Lectura: READ_EXTERNAL_STORAGE (no siempre explícito si solo se lee).

4.3. Permisos en tiempo de ejecución (Runtime Permissions)

  • Introducidos en Android 6.0 (API 23, Marshmallow).

  • Antes, todos los permisos se concedían al instalar la app. Ahora:

    • El usuario puede aceptar o denegar permisos durante la ejecución.

    • Ejemplo de comprobación:

      if (ContextCompat.checkSelfPermission(this, Manifest.permission.ACCESS_FINE_LOCATION)
        != PackageManager.PERMISSION_GRANTED) {
    ActivityCompat.requestPermissions(this,
        new String[]{Manifest.permission.ACCESS_FINE_LOCATION}, 1);
}

  • Esto aumenta la seguridad y protege la privacidad del usuario.

4.4. Conceptos adicionales de seguridad

  • Todas las aplicaciones Android funcionan en un sandbox, aisladas del resto del sistema y de otras apps.

  • Los permisos definen los canales de comunicación y recursos permitidos, evitando accesos no autorizados.

  • La combinación de manifest + runtime permissions asegura que la app solo pueda usar recursos explícitamente autorizados por el usuario.

5. Sensores y eventos de interacción

Android proporciona APIs para interactuar con sensores del dispositivo y capturar eventos de entrada del usuario, como gestos y toques.

5.1. Sensores

  • Acceso y gestión

    • SensorManager → clase principal para:

      • Obtener la lista de sensores disponibles.

      • Registrar o cancelar listeners para recibir eventos.

    • SensorEventListener → interfaz que define:

      • onSensorChanged(SensorEvent event) → notificación cuando cambia el sensor.

      • onAccuracyChanged(Sensor sensor, int accuracy) → notificación de cambios en precisión.

    • SensorEvent → objeto que contiene los valores generados por el sensor, la hora del evento y el sensor que lo generó.

  • Tipos de sensores

    • Movimiento: acelerómetro, giroscopio.

    • Posición: brújula, orientación.

    • Ambientales: luz, temperatura, presión.

    • Nota: “Sistemáticos” no es un tipo válido en Android.

  • Registro de sensores

    • Se registra un sensor especificando la frecuencia de actualización:

      sensorManager.registerListener(listener, sensor, SensorManager.SENSOR_DELAY_GAME);

5.2. Eventos táctiles

  • Captura de interacción táctil mediante MotionEvent:

    • Métodos principales:

      event.getX(); // coordenada X
event.getY(); // coordenada Y

    • Tipos de eventos:

      • MotionEvent.ACTION_DOWN → toque inicial.

      • MotionEvent.ACTION_MOVE → movimiento mientras se toca la pantalla.

      • MotionEvent.ACTION_UP → levantamiento del dedo.

  • Los eventos se manejan típicamente dentro de:

    @Override
public boolean onTouchEvent(MotionEvent event) {
    switch(event.getAction()) {
        case MotionEvent.ACTION_MOVE:
            // procesar movimiento
            break;
    }
    return true;
}

5.3. Uso práctico

  • Permite crear interfaces interactivas basadas en gestos y movimientos.

  • Facilita la integración con juegos, aplicaciones de realidad aumentada, o control de dispositivos físicos mediante sensores.

6. Redes y conectividad

Android ofrece APIs para gestionar conexiones de red, obtener información sobre su estado y capacidades, así como para interactuar con redes locales o remotas.

6.1. Clases principales

  1. Network

    • Representa una red concreta a la que el dispositivo está conectado o puede conectarse.

    • Se utiliza para identificar conexiones activas y asociarlas con recursos.

  2. NetworkCapabilities

    • Contiene información sobre capacidades de la red, como:

      • Tipo de transporte (Wi-Fi, celular, Ethernet).

      • Conectividad a Internet.

      • Validación y confiabilidad de la red.

  3. LinkProperties

    • Describe propiedades de la conexión, incluyendo:

      • Interfaces de red.

      • Direcciones IP asignadas.

      • DNS y rutas.

  4. ConnectivityManager

    • Permite comprobar la disponibilidad de redes y obtener Network o NetworkCapabilities.

    • Ejemplo:

      ConnectivityManager cm = (ConnectivityManager) getSystemService(Context.CONNECTIVITY_SERVICE);
Network network = cm.getActiveNetwork();
NetworkCapabilities caps = cm.getNetworkCapabilities(network);

6.2. Sockets

  • Orientados a conexión → TCP: transmisión confiable, ordenada, con control de errores.

  • No orientados a conexión → UDP: no garantiza fiabilidad ni orden, pero es más rápido para transmisión de datos en tiempo real.

6.3. Wi-Fi P2P y descubrimiento de servicios

  • Estado de dispositivos:

    • WifiP2pDevice.CONNECTED → indica que el dispositivo está conectado mediante Wi-Fi Direct.

  • Descubrimiento de servicios

    • Permite detectar servicios ofrecidos por otros dispositivos en la misma red sin necesidad de establecer una conexión directa.

6.4. Intents y transmisión de datos

  • Para pasar información entre componentes se utiliza Intent, que almacena datos en un Bundle:

    Intent intent = new Intent(this, OtraActivity.class);
intent.putExtra("clave", valor);
Bundle extras = intent.getExtras();

  • El Bundle funciona como un contenedor de datos clave-valor durante la transición entre componentes.

6.5. Aplicaciones prácticas

  • Determinar conectividad antes de hacer peticiones HTTP.

  • Envío de datos entre Activities.

  • Detección y utilización de servicios locales (como impresoras o otros dispositivos IoT) mediante Wi-Fi P2P.

7. Conectividad y servicios web

Android proporciona herramientas para interactuar con servicios remotos, descargar datos y mostrar contenido web dentro de la aplicación.

7.1. Descargas gestionadas

  • DownloadManager

    • Permite realizar descargas grandes gestionadas por el sistema operativo.

    • Gestiona colas, reintentos, notificaciones y comprobación de conectividad automáticamente.

    • Ejemplo de uso:

      DownloadManager downloadManager = (DownloadManager) getSystemService(DOWNLOAD_SERVICE);
Uri uri = Uri.parse("https://www.ejemplo.com/archivo.zip");
DownloadManager.Request request = new DownloadManager.Request(uri);
downloadManager.enqueue(request);

  • Permiso necesario:

    <uses-permission android:name="android.permission.INTERNET"/>

7.2. HTTP y códigos de estado

  • Para interactuar con servicios web, se utilizan peticiones HTTP.

  • Códigos de estado importantes:

    • 200 → OK, solicitud exitosa.

    • 301 → Redirección permanente.

    • 404 → Recurso no encontrado.

    • 500 → Error del servidor.

7.3. Google Maps

  • Para integrar mapas se utiliza Google Maps API.

  • Requisito: clave API asociada al proyecto para poder usar el servicio en pruebas o producción.

  • Permite mostrar mapas interactivos, marcadores, rutas y ubicación del usuario.

7.4. WebView

  • Componente que permite mostrar contenido web embebido dentro de la app.

  • Útil para:

    • Páginas HTML internas.

    • Integración de servicios web sin salir de la aplicación.

  • Uso básico:

    WebView webView = findViewById(R.id.webView);
webView.loadUrl("https://www.ejemplo.com");

7.5. Aplicaciones prácticas

  • Descargas de archivos y actualizaciones.

  • Visualización de contenido web dentro de la app.

  • Integración con servicios externos como mapas, APIs REST o mapas interactivos.

8. Herramientas y pruebas

Android Studio y las herramientas de línea de comandos permiten desarrollar, probar y depurar aplicaciones de manera eficiente.

8.1. Android Studio

  • IDE principal para desarrollo Android.

  • Funciones relacionadas con pruebas:

    • Crear y ejecutar pruebas unitarias o instrumentadas.

    • Ver resultados de pruebas directamente en el IDE.

    • Ejecutar y depurar aplicaciones en emuladores o dispositivos reales.

  • No incluye funciones irrelevantes como:

    • “Ocupa menos espacio los programas” (no es funcionalidad de prueba).

    • “Ventana Text Matrix” (no existe como herramienta de pruebas).

8.2. ADB (Android Debug Bridge)

  • Herramienta de línea de comandos para interactuar con el dispositivo o emulador.

  • Comandos útiles:

    • Enviar archivos al dispositivo/emulador:

      adb push archivo /sdcard/

    • Copiar archivos del dispositivo al host:

      adb pull /sdcard/archivo ./destino

    • Instalación de APKs, depuración y logcat.

8.3. Registro de eventos y depuración

  • Logcat permite visualizar mensajes de registro en tiempo real.

  • Clase principal: Log

    • Log.d("TAG", "Mensaje de depuración");

    • Log.i("TAG", "Información");

    • Log.e("TAG", "Error");

  • Útil para seguimiento de errores y comportamiento de la aplicación durante pruebas.

8.4. Aplicaciones prácticas

  • Probar la funcionalidad de la app antes de publicación.

  • Depurar errores y comprobar el flujo de datos y eventos.

  • Automatizar pruebas unitarias o instrumentadas desde Android Studio.

9. Recursos adicionales y documentación

Android ofrece documentación oficial y recursos de aprendizaje para facilitar el desarrollo de aplicaciones móviles de manera correcta y eficiente.

9.1. Android Developers

  • Es la documentación oficial de Google para Android.

  • Contiene guías, referencias y ejemplos de código para todas las funcionalidades de Android.

9.2. Formato y contenidos

  • La documentación utiliza principalmente Markdown para presentar:

    • Ejemplos de código.

    • Guías paso a paso.

    • Explicaciones de APIs y componentes.

  • Tipos de recursos:

    • Guías → explicaciones detalladas de conceptos y APIs.

    • Manuales de referencia → descripción técnica de clases, métodos y atributos.

    • Ejemplos de código → proyectos o fragmentos que ilustran su uso práctico.

  • Recursos que no forman parte de la documentación oficial:

    • Chat online de soporte directo.

    • Información sobre compatibilidad con iOS.

9.3. Temas principales

La documentación oficial cubre:

  • Sensores y eventos.

  • Audio y vídeo.

  • Contenido basado en web.

  • UI, layouts y componentes.

  • Conectividad y redes.

Esto proporciona a los desarrolladores una guía completa para implementar funcionalidades Android de forma consistente y correcta.

Resumen global – Unidad 02: Programación de aplicaciones para dispositivos móviles

1. Estructura y organización de proyectos

  • Código fuente en /src/main/java, paquetes nombrados con dominio invertido ( com.ejemplo.app ).

  • Recursos en /res: layouts, strings, drawables, estilos, colores.

  • Archivos clave:

    • R.java → IDs de recursos.

    • AndroidManifest.xml → permisos y componentes.

  • Gradle:

    • Proyecto → dependencias globales.

    • Módulo → dependencias del módulo.

  • APK → ejecutable Android, App Bundle → contiene todo el código y recursos para generar APK optimizados. Firma digital obligatoria.

  • Generación en Android Studio: Build → Build Bundle / APK → Build APK / Build App Bundle.

2. Actividades y UI

  • Activity → pantalla de la app, carga layout con setContentView(R.layout.nombre_layout).

  • WebView → mostrar contenido web embebido.

  • Toast → mensajes breves y no interactivos.

  • Componentes UI referenciados con R.id.nombreElemento y gestionados con listeners para eventos.

3. Manejo de recursos y almacenamiento

  • Acceso a recursos vía R.java (layouts, strings, drawables, colores).

  • Almacenamiento:

    • Archivos → FileInputStream.read().

    • SQLite:

      • getReadableDatabase(), getWritableDatabase().

      • rawQuery() → SELECT con Cursor.

      • execSQL() → ejecutar SQL sin retorno.

    • Permisos: WRITE_EXTERNAL_STORAGE para escritura externa.

  • Bundles → pasar datos entre Activities mediante Intent.putExtra() y getExtras().

4. Permisos y seguridad

  • Declarados en AndroidManifest.xml.

  • Tipos importantes:

    • INTERNET → acceso a red.

    • ACCESS_FINE_LOCATION → ubicación precisa.

    • ACCESS_COARSE_LOCATION → ubicación aproximada.

    • WRITE_EXTERNAL_STORAGE → almacenamiento externo.

  • Runtime Permissions (desde API 23, Marshmallow) permiten al usuario aceptar/denegar permisos durante ejecución.

  • Sandbox Android aísla apps; permisos definen recursos accesibles.

5. Sensores y eventos de interacción

  • Sensores:

    • SensorManager → gestionar sensores y listeners.

    • SensorEventListener → métodos onSensorChanged() y onAccuracyChanged().

    • SensorEvent → datos de cada evento.

  • Tipos de sensores: movimiento, posición, ambientales.

  • Registro de sensores: registerListener(listener, sensor, SENSOR_DELAY_GAME).

  • Eventos táctiles: MotionEvent ( getX(), getY(), ACTION_MOVE, ACTION_DOWN, ACTION_UP ).

6. Redes y conectividad

  • Clases clave:

    • Network → representa red activa.

    • NetworkCapabilities → capacidades (transporte, internet, validación).

    • LinkProperties → propiedades de conexión (IP, rutas, DNS).

    • ConnectivityManager → verificar redes disponibles.

  • Sockets: TCP → confiable; UDP → no orientado a conexión, no garantiza orden.

  • Wi-Fi P2P:

    • WifiP2pDevice.CONNECTED → estado de conexión.

    • Descubrimiento de servicios → detectar servicios sin conectar.

  • Intents → pasar datos entre componentes mediante Bundle.

7. Conectividad y servicios web

  • DownloadManager → descargas grandes gestionadas por el sistema.

  • HTTP:

    • Códigos: 200 (OK), 301 (redirección permanente), 404 (no encontrado), 500 (error servidor).

  • Google Maps → requiere clave API.

  • WebView → mostrar contenido web interno.

  • Uso práctico: descargar archivos, mostrar contenido web, integrar servicios externos.

8. Herramientas y pruebas

  • Android Studio → crear, ejecutar y depurar pruebas.

  • ADB:

    • adb push → enviar archivo al dispositivo/emulador.

    • adb pull → copiar archivo al host.

  • Logcat:

    • Clase Log → Log.d(), Log.i(), Log.e().

    • Permite seguimiento de errores y comportamiento de la app.

9. Recursos adicionales y documentación

  • Android Developers → documentación oficial de Google.

  • Formato Markdown para guías, ejemplos y referencias.

  • Contenido: guías, manuales de referencia, ejemplos de código.

  • Temas principales: sensores, audio y vídeo, contenido web, UI, conectividad.

  • No incluye compatibilidad con iOS ni chat online.
