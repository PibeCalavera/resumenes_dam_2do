1. Ficheros en Java (Parte 1: Fundamentos y manejo básico)

1.1. Tipos de ficheros

Java distingue principalmente:

Tipo de fichero Contenido Clases típicas
Texto Caracteres ( char ) FileReader, FileWriter, BufferedReader, BufferedWriter
Binario Bytes ( byte ) FileInputStream, FileOutputStream, DataInputStream, DataOutputStream
Acceso aleatorio Binario con puntero RandomAccessFile

1.2. Clase File (java.io)

  • Representa la ruta a un fichero o directorio, no su contenido.

  • Se usa para comprobar existencia, tipo, tamaño, listar directorios, crear/eliminar ficheros.

Métodos clave:

File f = new File("ruta/archivo.txt");

f.exists();        // true si existe
f.isFile();        // true si es fichero
f.isDirectory();   // true si es directorio
f.length();        // tamaño en bytes
f.getName();       // nombre del fichero
f.getPath();       // ruta

⚠️ Recuerda: usar / es más portable que \\.

1.3. Lectura y escritura de ficheros de texto

Lectura

Método Clase Qué hace
read() FileReader Lee un carácter. Devuelve int o -1 si EOF
read(char[]) FileReader Lee varios caracteres en un array
readLine() BufferedReader Lee una línea completa ( null = EOF)

Ejemplo:

BufferedReader br = new BufferedReader(new FileReader("archivo.txt"));
String linea;
while ((linea = br.readLine()) != null) {
    System.out.println(linea);
}
br.close();

Escritura

  • FileWriter → escritura básica

  • BufferedWriter → más eficiente, permite newLine()

BufferedWriter bw = new BufferedWriter(new FileWriter("salida.txt"));
bw.write("Hola mundo");
bw.newLine();
bw.close();

1.4. Lectura y escritura de ficheros binarios

Clases básicas

  • Lectura: FileInputStream

  • Escritura: FileOutputStream

  • Tipos primitivos: DataInputStream, DataOutputStream

Métodos importantes

Clase Método Función
DataOutputStream writeInt(), writeBoolean(), writeUTF() Escribe primitivos y cadenas UTF
DataInputStream readInt(), readBoolean(), readUTF() Lee primitivos y cadenas UTF

Ejemplo:

DataOutputStream dos = new DataOutputStream(new FileOutputStream("datos.bin"));
dos.writeInt(123);
dos.writeUTF("Hola");
dos.close();

1.5. Acceso aleatorio: RandomAccessFile

Permite leer/escribir en cualquier posición sin recorrer el fichero secuencialmente.

Métodos clave:

RandomAccessFile raf = new RandomAccessFile("archivo.bin", "rw");

raf.seek(10);             // mueve el puntero
long pos = raf.getFilePointer(); // obtiene la posición actual
raf.writeInt(50);         // escribe un int en la posición actual
raf.readInt();            // lee un int desde la posición actual
raf.length();             // tamaño del fichero
raf.close();

1.6. Excepciones y cierre de recursos

  • Operaciones con ficheros lanzan:

    • IOException

    • FileNotFoundException

  • Siempre cerrar ficheros:

try (FileReader fr = new FileReader("archivo.txt")) {
    // lectura
} // se cierra automáticamente (try-with-resources)

1.7. Conceptos clave a memorizar

  • EOF → End Of File ( -1 para read(), null para readLine() )

  • Primitivos vs Wrapper → int vs Integer, boolean vs Boolean

  • Texto vs Binario → tipo de dato + clase correcta

  • Acceso secuencial vs aleatorio → FileReader / DataInputStream vs RandomAccessFile

  • Flujos (Stream) → InputStream / OutputStream → bytes, Reader / Writer → caracteres

2. Ficheros HTML, XML, JSON y datos web en Java

2.1. XML y DOM

Clases clave

Clase Función
Document Representa el documento XML completo en memoria
Element Representa un elemento XML dentro del documento
Attr Representa un atributo de un elemento
Node Representa cualquier nodo del DOM (elemento, texto, comentario, etc.)

Ejemplo de uso básico

DocumentBuilderFactory dbf = DocumentBuilderFactory.newInstance();
DocumentBuilder db = dbf.newDocumentBuilder();
Document doc = db.parse(new File("archivo.xml")); // carga XML
Element root = doc.getDocumentElement();          // obtiene nodo raíz

2.2. Transformaciones XSL

Clases clave

  • TransformerFactory → fabrica Transformer

  • Transformer → aplica plantilla XSL

  • Source → origen de los datos (archivo XML, DOM)

  • Result → destino de la transformación (archivo, stream, DOM)

Método principal

transformer.transform(Source xmlSource, Result outputTarget);

Ejemplo

TransformerFactory tf = TransformerFactory.newInstance();
Transformer transformer = tf.newTransformer(new StreamSource("plantilla.xsl"));
Source source = new StreamSource("entrada.xml");
Result result = new StreamResult(new File("salida.xml"));
transformer.transform(source, result);

  • Convierte un documento XML en otro XML (o HTML) según la plantilla XSL.

2.3. JSON en Java (org.json)

Clases clave

Clase Función
JSONObject Representa un objeto JSON (clave-valor)
JSONArray Representa un array JSON
HTTP Convierte entre JSON ↔ texto de encabezado HTTP
Cookies Convierte entre JSON ↔ texto de cookies HTTP
CDL Convierte entre JSON ↔ texto delimitado por comas (CSV)

Ejemplos

JSON ↔ HTTP

JSONObject json = new JSONObject();
json.put("name", "Juan");
String header = HTTP.toString(json);           // JSON → HTTP
JSONObject obj = HTTP.toJSONObject(header);    // HTTP → JSON

JSON ↔ Cookies

String cookieText = Cookies.toString(json);           // JSON → cookie
JSONObject obj2 = Cookies.toJSONObject(cookieText);  // cookie → JSON

JSON ↔ CSV

JSONArray ja = new JSONArray();
ja.put(new JSONObject().put("name", "Ana").put("age", 25));
String csv = CDL.toString(ja);                  // JSON → CSV
JSONArray ja2 = CDL.rowToJSONArray(csv);       // CSV → JSON

2.4. Escritura y lectura de XML / JSON en ficheros

XML

  • DOM para manipulación en memoria ( Document )

  • Transformer para escritura de Document a fichero XML

  • DOMSource → origen, StreamResult → destino

JSON

  • JSONObject y JSONArray → datos en memoria

  • FileWriter / BufferedWriter para guardar JSON en fichero

try (FileWriter fw = new FileWriter("datos.json")) {
    fw.write(json.toString(4)); // 4 = indentación
}

  • FileReader / BufferedReader → lectura desde fichero y luego new JSONObject(reader) o new JSONArray(reader)

2.5. Conceptos clave para examen

  1. DOM → manipulación de XML en memoria

  2. Transformer + XSL → transformar XML a XML/HTML

  3. JSONObject / JSONArray → representación de datos JSON

  4. Clases auxiliares org.json:

    • HTTP → JSON ↔ headers HTTP

    • Cookies → JSON ↔ cookies HTTP

    • CDL → JSON ↔ CSV

  5. Guardar en fichero:

    • XML → Transformer.transform(DOMSource, StreamResult)

    • JSON → FileWriter.write(json.toString())

  6. Leer desde fichero:

    • XML → DocumentBuilder.parse(File)

    • JSON → new JSONObject(Reader) o new JSONArray(Reader)

3. Manejo de Excepciones en Java

3.1. Concepto de excepción

  • Una excepción es un evento anómalo que ocurre durante la ejecución del programa y interrumpe el flujo normal.

  • En Java, las excepciones son objetos que heredan de Throwable.

3.2. Jerarquía principal

java.lang.Object
   └── java.lang.Throwable
         ├── java.lang.Error       // errores graves, no manejables normalmente
         └── java.lang.Exception   // excepciones que se pueden capturar
               ├── RuntimeException  // unchecked (no obligatorio capturar)
               └── Checked exceptions  // obligatorias de capturar o declarar

  • Checked exceptions: deben ser capturadas o declaradas con throws Ej.: IOException, FileNotFoundException

  • Unchecked exceptions: no obligatorias de capturar ( RuntimeException y subclases) Ej.: NullPointerException, ArithmeticException

3.3. Captura de excepciones

Sintaxis básica try-catch:

try {
    // código que puede lanzar excepción
} catch (TipoDeExcepcion e) {
    // manejo de la excepción
} finally {
    // código que se ejecuta siempre (opcional)
}

  • try → bloque protegido

  • catch → captura la excepción específica

  • finally → se ejecuta siempre, útil para liberar recursos (cerrar ficheros, streams)

Ejemplo:

try {
    FileReader fr = new FileReader("archivo.txt");
} catch (FileNotFoundException e) {
    System.out.println("Archivo no encontrado: " + e.getMessage());
} finally {
    System.out.println("Bloque finally ejecutado");
}

3.4. Propagación de excepciones

  • Se utiliza la palabra clave throws en la firma de un método:

public void leerArchivo() throws IOException {
    FileReader fr = new FileReader("archivo.txt");
}

  • Indica que el método puede lanzar una excepción y el llamador debe manejarla.

3.5. Métodos importantes de las excepciones

Todos los objetos que heredan de Throwable tienen métodos útiles para obtener información:

Método Función
getMessage() Devuelve mensaje descriptivo de la excepción
printStackTrace() Imprime traza completa de la excepción (dónde ocurrió)
toString() Representación breve de la excepción

Ejemplo:

try {
    int a = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println(e.getMessage());
    e.printStackTrace();
}

3.6. Excepciones comunes en ficheros y datos

Excepción Cuándo ocurre
IOException Error de entrada/salida general
FileNotFoundException Fichero no encontrado
EOFException Fin de fichero alcanzado en binarios
ParseException Error al leer o parsear datos (por ejemplo XML o JSON)

3.7. Buenas prácticas

  1. Capturar solo lo necesario → no capturar Exception genérico si puedes usar una subclase específica

  2. Usar finally o try-with-resources para cerrar ficheros y streams

  3. Documentar las excepciones lanzadas con @throws en Javadoc

  4. Evitar excepciones silenciosas → siempre registrar o informar del error

3.8. Ejemplo completo combinando conceptos

/**
 * Lee el contenido de un fichero.
 * @param nombreArchivo ruta del archivo
 * @return contenido como String
 * @throws IOException si ocurre un error de lectura
 */
public String leerArchivo(String nombreArchivo) throws IOException {
    StringBuilder contenido = new StringBuilder();
    try (BufferedReader br = new BufferedReader(new FileReader(nombreArchivo))) {
        String linea;
        while ((linea = br.readLine()) != null) {
            contenido.append(linea).append("\n");
        }
    }
    return contenido.toString();
}

  • Combina try-with-resources, captura de excepciones, y documentación Javadoc.

4. Documentación en Java (Javadoc)

4.1. Qué es Javadoc

  • Javadoc es la herramienta oficial de Java para generar documentación HTML a partir de comentarios en el código fuente.

  • Se basa en comentarios especiales que comienzan con /** ... */.

  • Permite describir clases, métodos, parámetros, excepciones y enlaces a otros recursos.

/**
 * Esta clase representa un ejemplo de Javadoc.
 */
public class Ejemplo { }

4.2. Comentarios Javadoc

  • Comienzan con /** y terminan con */.

  • Pueden incluir texto descriptivo y etiquetas especiales ( @ ) para estructurar la información.

Ejemplo:

/**
 * Calcula la suma de dos números.
 */
public int sumar(int a, int b) { return a + b; }

4.3. Etiquetas más importantes

Etiqueta Uso
@param Describe cada parámetro de un método
@return Describe el valor de retorno de un método
@throws Describe excepciones que el método puede lanzar
@see Referencia a otra clase, método o recurso relacionado
@deprecated Indica que el método o clase está obsoleto
@author Autor del código
@version Versión de la clase
@since Versión de Java o del proyecto desde la cual existe el elemento

4.4. Sintaxis y ejemplos

Documentar parámetros y retorno

/**
 * Calcula la suma de dos números enteros.
 * @param a primer número
 * @param b segundo número
 * @return la suma de a y b
 */
public int sumar(int a, int b) { return a + b; }

Documentar excepciones

/**
 * Lee un archivo y devuelve su contenido.
 * @param nombreArchivo ruta del archivo
 * @return contenido del archivo
 * @throws IOException si ocurre un error de lectura
 */
public String leerArchivo(String nombreArchivo) throws IOException { ... }

Referencias a otros métodos o clases

/**
 * Calcula la suma de dos números.
 * @see Math#addExact(int,int)
 */
public int sumar(int a, int b) { return a + b; }

Marcar métodos obsoletos

/**
 * @deprecated usar sumarExacto(int,int) en su lugar
 */
public int sumar(int a, int b) { return a + b; }

4.5. Símbolo común

  • Todas las etiquetas Javadoc comienzan con @.

  • Ejemplos: @param, @return, @throws, @see, @deprecated

4.6. Buenas prácticas

  1. Documentar todos los métodos públicos y sus parámetros

  2. Indicar claramente excepciones con @throws

  3. Usar @see para vincular métodos relacionados o clases útiles

  4. Marcar como obsoletos con @deprecated y sugerir alternativa

  5. Mantener consistencia y claridad en los comentarios

4.7. Ejemplo completo combinando etiquetas

/**
 * Calcula la suma de dos enteros positivos.
 * @param a primer número (debe ser positivo)
 * @param b segundo número (debe ser positivo)
 * @return la suma de a y b
 * @throws IllegalArgumentException si a o b son negativos
 * @see Math#addExact(int,int)
 * @deprecated usar sumarExacto(int,int) en su lugar
 */
public int sumar(int a, int b) throws IllegalArgumentException {
    if (a < 0 || b < 0) throw new IllegalArgumentException();
    return a + b;
}

  • Incluye: descripción, parámetros, retorno, excepción, referencia y obsolescencia.

  • Perfecto como ejemplo de examen.
