1. Introducción a los Sistemas de Gestión Empresarial

1.1 Definición ERP:

  • ERP (Enterprise Resource Planning): sistema integrado que gestiona los procesos empresariales en áreas como finanzas, producción, ventas, compras, RRHH, BI y proyectos.

  • Objetivo: optimizar recursos, automatizar procesos y centralizar información.

1.2 Definición CRM:

  • CRM (Customer Relationship Management): sistema que gestiona la relación con clientes, automatiza ventas, marketing y servicios, y registra interacciones para mejorar la atención.

1.3 Relación ERP-CRM:

  • Muchos ERP incorporan módulos CRM, pero un ERP completo no sustituye todas las funcionalidades de un CRM ⚠️.

  • ERP + CRM = visión integral de la empresa y gestión completa del cliente y operaciones.

2. Tipos de licencias de software y contratos

2.1 Licencia de software: definición

  • Contrato legal entre el desarrollador y el usuario, que establece derechos y obligaciones sobre el uso, distribución y modificación del software.

  • Forma parte de los EULA (End User License Agreement) ⚠️: documento clave que se menciona en examen.

2.2 Tipos principales de licencias

Tipo Características Ejemplo ERP/CRM
Propietario Uso restringido, sin acceso al código fuente, modificaciones prohibidas Microsoft Dynamics 365 ERP/CRM
Freeware Uso gratuito, redistribución permitida, modificaciones restringidas ⚠️ Versiones gratuitas de Odoo (community)
Shareware Evaluación gratuita por tiempo limitado, pago posterior Versiones trial de SAP, Dynamics
Software libre / Open Source Libertad para usar, modificar, distribuir, acceso al código fuente ⚠️ Odoo, ERPNext
SaaS Software como servicio, accesible vía web, mantenimiento y actualización gestionados por proveedor Dynamics 365 Online, Salesforce

2.3 Conceptos clave

  • Redistribución: derecho a compartir el software.

  • Modificación: capacidad de alterar el código fuente.

  • Derechos de autor / propiedad intelectual: protegen la titularidad del software.

  • ⚠️ En examen, preguntar por Freeware o Shareware, la opción correcta puede depender de si se permite redistribución o modificación.

3. Arquitectura e instalación de ERP/CRM

3.1 Arquitectura cliente-servidor

  • Definición: separación entre cliente (interfaz de usuario) y servidor (lógica de negocio y base de datos).

  • Tipos:

    1. 2 capas: Cliente ↔ Servidor BD

      • Ventaja: simple, menos mantenimiento.

      • Desventaja: menos escalable.

    2. 3 capas: Cliente ↔ Servidor de aplicación ↔ Servidor BD ⚠️

      • Ventaja: escalable, permite separar lógica de presentación, negocio y datos.

Diagrama simplificado:

Cliente (UI)
    │
    ▼
Servidor de Aplicación (ERP/CRM)
    │
    ▼
Servidor Base de Datos (SQL Server / MySQL / PostgreSQL)

3.2 Parámetros principales en la instalación

  • Idioma y localización: moneda, formato de fecha, zona horaria.

  • Base de datos: tipo (SQL Server, MySQL, PostgreSQL), usuario y contraseña.

  • Credenciales de administrador: acceso y permisos.

  • Red y puertos: configuración de comunicación cliente-servidor.

  • Módulos a instalar: ERP (finanzas, RRHH, producción…), CRM (ventas, marketing, servicio).

3.3 Sistemas gestores de bases de datos utilizados

ERP/CRM SGBD
Microsoft Dynamics 365 SQL Server ⚠️
Odoo (Community) PostgreSQL
Odoo (antiguo / MySQL) MySQL
ERPNext MariaDB / MySQL

3.4 Actualizaciones del sistema

  • Automáticas ⚠️: predeterminadas en la instalación, el sistema se mantiene actualizado sin intervención.

  • Manuales: requieren que el administrador aplique cada actualización.

  • Regla examen: marcar Automáticas salvo que se indique lo contrario.

3.5 Errores comunes en instalación

  • No configurar correctamente usuario/contraseña de BD.

  • Confundir arquitectura 2 vs 3 capas y desplegar componentes en lugares incorrectos.

  • Ignorar la correcta selección de idioma y localización.

  • Deshabilitar actualizaciones automáticas sin plan de mantenimiento.

4. Entornos de trabajo ERP/CRM

4.1 Definición general

  • Un entorno de trabajo es un conjunto de recursos (software, base de datos, aplicaciones) diseñado para un propósito concreto durante la implantación de un ERP/CRM.

  • ⚠️ Concepto clave para examen: el profesor asocia “implantación” con Explotación, aunque técnicamente la validación del usuario final ocurre en UAT/Preproducción.

4.2 Tipos de entornos

Entorno Propósito Componentes clave Notas examen
Desarrollo Codificación, parametrización, pruebas internas IDE, servidor de control de versiones, BD de desarrollo, binarios, software adicional Principalmente usado por desarrolladores
Pruebas / Testing Validar funcionalidad y flujos antes de UAT BD de pruebas, instalación de aplicación, software necesario ⚠️ UAT puede formar parte de este entorno
UAT / Preproducción Validación por usuarios finales Copia de BD de pruebas, ERP/CRM configurado, software necesario Objetivo: que el cliente acepte los procesos
Explotación / Producción Sistema operativo final Servidor de aplicación, BD de producción, software adicional ⚠️ Según el temario, “implantación” = explotación

4.3 Componentes de cada entorno

Desarrollo

  • IDE / editor de código

  • Servidor de control de versiones

  • Base de datos de desarrollo

  • Binarios de la aplicación

  • Software adicional de soporte

Pruebas / UAT

  • Base de datos de pruebas

  • Aplicación configurada para testing

  • Software adicional necesario

Explotación

  • Base de datos de producción

  • Aplicación en entorno operativo

  • Software adicional requerido para funcionamiento real

4.4 Errores comunes

  • Confundir UAT con Explotación ⚠️

  • Realizar pruebas directamente en la BD de producción

  • No replicar correctamente la configuración del entorno de desarrollo en pruebas

5. Módulos principales de ERP y CRM

5.1 Módulos ERP

  1. Administración financiera

    • General: planes de cuentas, contabilidad básica, presupuestos, control de cambios.

    • Activos fijos: gestión de bienes materiales (edificios, vehículos, seguros…).

    • Tesorería: operaciones financieras con bancos y cajas de ahorro.

  2. Administración de la cadena de suministros (SCM)

    • Ventas, compras, inventario/almacén.

    • Control automático del stock y repercusiones contables.

  3. Fabricación / Producción

    • Control de materiales, estado de fabricación, productos finalizados.

    • Órdenes de producción, rutas y listas de materiales (BOM).

  4. Administración de proyectos

    • Gestión de recursos, planificación y seguimiento de tareas.

  5. Business Intelligence (BI)

    • Power BI, análisis de datos, informes y KPIs.

  6. Recursos Humanos (RRHH)

    • Gestión de roles, accesos, tipo de contrato, datos de empleados.

5.2 Módulos CRM (Microsoft Dynamics CRM)

Ventas (Sales) ⚠️

  • Soporte al ciclo de vida de ventas:

    • Buscar y calificar clientes potenciales (Leads).

    • Gestionar oportunidades y fases de cierre.

    • Seguimiento de comunicaciones vendedor-cliente.

    • Mantener base de datos de productos.

  • Registros utilizados: Leads, Oportunidades, Ofertas, Pedidos, Facturas, Cuentas, Contactos, Productos.

Marketing

  • Actividades: campañas exprés, campañas, respuestas de campaña, listas de marketing.

  • Campañas rápidas → actividad única; campañas → múltiples actividades y seguimiento financiero.

  • Respuestas pueden convertirse en Leads/Oportunidades ⚠️.

Servicio (Service) ⚠️

  • Registro de casos, contratos, derechos de servicio.

  • Knowledge Base: FAQs, resolución de problemas, prácticas recomendadas.

  • Programación de recursos y citas mediante motor de planificación.

  • Cola de servicios y enrutamiento para mejorar la tramitación de solicitudes.

5.3 Glosario y precisiones

  • Lead / Cliente potencial: persona u organización a calificar.

  • Oportunidad: venta potencial a una cuenta o contacto.

  • Cuenta / Contacto: pueden ser cliente o prospecto, no necesariamente cliente real.

5.4 Errores comunes

  • Confundir Lead con Oportunidad ⚠️

  • Asumir que Cuenta = Cliente

  • No distinguir las funciones entre módulos: Marketing ≠ Ventas ≠ Servicio

6. Ejecución de Dynamics 365 Business Central y CRM + autenticación

6.1 Opciones de ejecución

Sistema Opciones de ejecución según temario Notas examen
Dynamics 365 Business Central Web y aplicación de escritorio Predeterminado para clientes locales o en nube híbrida
Dynamics 365 CRM ⚠️ Solo Web según temario Aunque existe cliente de escritorio, el examen espera marcar solo web

Observaciones:

  • Business Central permite acceso multiplataforma (Web/Desktop).

  • CRM se utiliza principalmente vía navegador, especialmente en ejercicios del temario.

  • Ejecución en web facilita SaaS y actualizaciones automáticas.

6.2 Autenticación integrada

  • Windows / Active Directory ⚠️: usada en instalaciones locales de Business Central y CRM.

  • Permite que usuarios accedan con credenciales de dominio sin nuevas cuentas.

  • Facilita control de permisos y roles.

6.3 Errores comunes

  • Marcar “Web + escritorio” para CRM → incorrecto según temario.

  • Confundir autenticación por BD vs autenticación integrada.

  • No comprobar permisos de Active Directory antes de instalar el cliente.

7. Actualizaciones y mantenimiento de ERP/CRM

7.1 Tipos de actualizaciones

  1. Automáticas ⚠️

    • Predeterminadas en la instalación según temario.

    • Se aplican sin intervención del administrador.

    • Mantienen la seguridad y funcionamiento correcto del sistema.

  2. Manuales

    • Requieren intervención del administrador.

    • Se activan mediante aviso de actualización.

    • Menos recomendable en entornos productivos, salvo control de cambios.

7.2 Recomendaciones de instalación y mantenimiento

  • Mantener actualizaciones automáticas activadas salvo que se trabaje en entornos específicos de pruebas.

  • Realizar copias de seguridad periódicas de la base de datos antes de aplicar actualizaciones.

  • Comprobar compatibilidad de actualizaciones con módulos personalizados o integraciones externas.

7.3 Errores comunes

  • Deshabilitar actualizaciones automáticas en producción ⚠️ → riesgo de seguridad y fallos.

  • Aplicar actualizaciones directamente en la BD de explotación sin pruebas.

  • Ignorar dependencias de módulos antes de actualizar.

8. Ciclo de implantación ERP/CRM y relación con entornos

8.1 Fases del ciclo de implantación

  1. Desarrollo

    • Creación y personalización de funcionalidades.

    • Pruebas internas del equipo de desarrollo.

  2. Pruebas / Testing

    • Validación funcional y técnica del ERP/CRM.

    • Permite detectar errores antes de involucrar usuarios finales.

  3. UAT / Preproducción ⚠️

    • Objetivo: que los usuarios finales validen los procesos y acepten la solución antes de pasar a producción.

    • Ambiente lo más cercano a producción.

    • Se utiliza para verificar flujos de negocio, integraciones y permisos.

  4. Explotación / Producción ⚠️

    • Entorno donde se implanta el ERP/CRM.

    • Según el temario, “implantación” se asocia con explotación.

    • Base de datos y aplicaciones finales operativas.

8.2 Componentes clave de cada fase

Fase Componentes principales Responsables
Desarrollo IDE, control de versiones, BD de desarrollo, binarios, herramientas adicionales Desarrolladores
Pruebas BD de pruebas, aplicación configurada, software necesario Equipo de QA / técnicos
UAT / Preproducción BD de prueba replicada, aplicación, configuraciones finales Usuarios finales / cliente
Explotación BD de producción, aplicación, servicios, seguridad Administradores / IT

8.3 Errores comunes

  • Confundir UAT con Explotación ⚠️

  • Realizar pruebas directamente en la BD de producción

  • No validar permisos o configuraciones antes de pasar a explotación
