1. Definición de campos

1.1 Concepto general (común ERP–CRM)

Clasificación general:

  • Texto → almacenamiento de caracteres.

  • Numéricos → cantidades, importes, valores contables.

  • Fecha / Fecha y hora → gestión temporal de operaciones.

Esto es base conceptual para:

  • Formularios (6)

  • Tablas (4)

  • Auditoría a nivel de campo (10.1)

  • Trazabilidad y movimientos (8)

Es un bloque fundacional, aunque no tenga muchas preguntas directas.

2. Tablas y vistas del sistema

2.1 Concepto de tabla

Una tabla es la estructura básica de almacenamiento de información en un sistema ERP o CRM.

En ella se organizan los datos en:

  • Campos (columnas) → Definen el tipo de información que se almacena.

  • Registros (filas) → Representan cada instancia concreta de información.

Ejemplo: Una tabla de Clientes puede contener campos como:

  • Código

  • Nombre

  • Dirección

  • Teléfono

  • Fecha de alta

Cada cliente concreto será un registro dentro de esa tabla.

Las tablas constituyen la capa de persistencia de datos del sistema, es decir, el lugar donde la información queda almacenada de forma estructurada.

2.2 Tablas en Microsoft Dynamics Nav / Business Central

En entornos como Microsoft Dynamics Nav y Microsoft 365 Business Central, los datos se organizan mediante:

  • Tablas definidas técnicamente en el sistema.

  • Campos asociados a cada tabla.

  • Relaciones entre tablas mediante claves.

Las tablas permiten:

  • Almacenar datos empresariales (clientes, proveedores, productos, asientos contables…).

  • Servir de base para consultas (Queries).

  • Alimentar formularios e informes.

2.3 Entidades en Microsoft Dynamics 365 CRM

En Microsoft Dynamics 365 CRM, el concepto equivalente a tabla es la Entidad.

Una entidad:

  • Contiene campos.

  • Almacena registros.

  • Representa un objeto de negocio (Cuenta, Contacto, Caso, Oportunidad, etc.).

Por tanto:

Tabla (Nav/Business Central) ≈ Entidad (Dynamics 365 CRM)

La diferencia es principalmente terminológica y de entorno de configuración, pero conceptualmente cumplen la misma función: estructurar y almacenar datos.

2.4 Vistas del sistema

Una vista es una forma de mostrar un subconjunto de registros de una tabla o entidad según determinados criterios.

Características:

  • No modifica la estructura de la tabla.

  • No altera los datos almacenados.

  • Permite filtrar y organizar la visualización.

Ejemplo:

  • Vista “Clientes activos”.

  • Vista “Facturas pendientes”.

  • Vista “Casos abiertos”.

Las vistas facilitan la consulta operativa de la información sin afectar a la base de datos.

2.5 Importancia dentro del sistema

Las tablas o entidades son la base sobre la que se construyen:

  • Las consultas de acceso a datos.

  • Los formularios de entrada y visualización.

  • Los informes.

  • Los procesos de auditoría.

  • Los análisis y cuadros de mando.

Sin tablas correctamente definidas, no es posible organizar ni consultar la información de forma estructurada.

3. Consultas de acceso a datos

3.1 Concepto de consulta (Query)

Una consulta es un objeto que permite acceder y recuperar información almacenada en las tablas o entidades del sistema ERP/CRM.

Permite:

  • Filtrar registros según criterios específicos.

  • Ordenar los datos de manera útil para el análisis.

  • Relacionar información de diferentes tablas o entidades.

Importante: Una consulta no es una interfaz visual ni un informe imprimible; su función principal es proporcionar conjuntos de datos precisos para posteriores procesos o análisis.

3.2 Tipos de objetos en Business Central / Dynamics 365 CRM

Para evitar confusiones comunes en examen, conviene diferenciar tres objetos clave:

Objeto Función principal
Query Acceso y recuperación de datos de tablas o entidades
Page Visualización de datos al usuario, interfaz de entrada y consulta
Report Generación de informes imprimibles o exportables (PDF, Excel)
⚠️ Esta distinción es un punto frecuente de examen.

4. Interfaces de entrada de datos y formularios

4.1 Interfaz de entrada de datos

La interfaz de entrada de datos es la capa que permite a los usuarios introducir y consultar información en el sistema ERP o CRM.

Características principales:

  • Muestra de listados de registros para facilitar la visualización.

  • Permite navegar y seleccionar registros sin alterar la estructura de las tablas.

  • Su aspecto puede variar según el sistema (Business Central o Dynamics 365 CRM).

4.2 Formularios

Los formularios son la representación visual de los datos contenidos en tablas o entidades.

Componentes estándar de todos los formularios:

  • Encabezado (Header): Información clave del registro.

  • Cuerpo (Body): Campos detallados organizados en secciones.

  • Pie (Footer): Resumen de información o botones de acción.

4.3 Secciones

Las secciones dentro de un formulario permiten:

  • Agrupar campos relacionados para una organización lógica.

  • Mejorar la usabilidad y la navegación del usuario.

Ejemplo: En un formulario de Cliente:

  • Sección 1 → Datos de contacto

  • Sección 2 → Información fiscal

  • Sección 3 → Dirección

4.4 Relación con otros objetos

  • Page: Objeto de la interfaz que permite mostrar formularios y listados.

  • Campos: Son los elementos que se muestran y editan en los formularios.

  • Tablas/Entidades: Almacenan los datos que se consultan y modifican desde los formularios.

⚠️ Punto frecuente de examen: No confundir Page (interfaz) con Query (acceso a datos) ni Report (informes).

5. Informes y listados de la aplicación

5.1 Tipos de informes

En un sistema ERP/CRM, los informes y listados permiten consultar, presentar y analizar los datos almacenados.

  • Report: Objeto que genera informes imprimibles o exportables (PDF, Excel).

  • Listados: Visualización rápida de datos dentro del sistema para consultas operativas.

⚠️ Punto frecuente de examen: Report es para imprimir/exportar; no confundir con Query (acceso a datos) ni con Page (interfaz visual).

5.2 Esquemas de cuentas

Definición: Un esquema de cuentas es un conjunto de reglas que automatiza la creación de registros contables con estructura fija, como nóminas o impuestos.

Un esquema de cuentas sirve para:

  1. Organizar las cuentas del plan de cuentas

  2. Generar informes financieros

  3. Comparar diferentes esquemas mediante fórmulas

  4. Crear informes personalizados

Componentes de un esquema de cuentas:

  • Nombre del esquema

  • Líneas del esquema

  • Plantilla de columnas

Varias preguntas de examen confirman que todas estas opciones son correctas.

5.3 Plantillas de columnas

  • Definen la estructura de los informes.

  • Permiten uniformidad en la comparación de esquemas.

  • No son informes en sí, sino un formato para generar informes.

5.4 Comparación de esquemas

  • Las fórmulas permiten comparar dos o más esquemas de cuentas.

  • Esto posibilita crear informes financieros personalizados.

  • Importante para detectar diferencias en configuraciones o resultados contables.

5.5 Conclusión

Este bloque es el núcleo central de la unidad, ya que combina:

  • Organización de cuentas mediante esquemas.

  • Extracción y presentación de datos con Reports.

  • Uso de plantillas y fórmulas para comparaciones y análisis financiero.

⚠️ Puntos críticos de examen: diferencia Report vs plantilla, finalidad de los esquemas de cuentas, y capacidad de comparación mediante fórmulas.

6. Gestión operativa

La gestión operativa se encarga de los procesos diarios de la empresa dentro del ERP/CRM, incluyendo productos, compras, ajustes contables, movimientos de transacciones y ventas/clientes.

Punto clave de examen:

  • La ficha de cliente se actualiza mediante transacciones o actualización de datos en el sistema.

Otros procesos (productos, compras, ajustes contables, movimientos) se mencionan para contextualizar la operativa, aunque no suelen preguntarse en examen.

⚠️ Resumen rápido: comprender que todas estas operaciones se registran en tablas y se reflejan en formularios, informes y auditorías.

7. Gráficos

Los gráficos son herramientas visuales que permiten representar de manera rápida y comprensible la información almacenada en el ERP/CRM.

Son útiles para:

  • Analizar tendencias y patrones de negocio.

  • Presentar información de manera clara y personalizada.

  • Apoyar la toma de decisiones a partir de datos consolidados.

7.1 Tipos de gráficos

  • Gráficos del sistema: Predefinidos por el ERP/CRM, muestran información estándar como ventas, stock o facturación.

  • Gráficos personales: Configurados por el usuario según consultas o necesidades específicas.

7.2 Características principales

  • Reflejan datos provenientes de tablas / entidades.

  • Se apoyan en queries o informes como fuente de datos.

  • Facilitan la interpretación de grandes volúmenes de información.

8. Herramientas de monitorización y auditoría

8.1 Auditoría en Dynamics 365 CRM (actualizado)

La auditoría permite rastrear los cambios realizados en los datos y la configuración del sistema.

Niveles de auditoría:

  1. Nivel de organización

    • Debe estar habilitado primero para que cualquier otra auditoría funcione.

    • Audita cambios a nivel del sistema completo, incluyendo configuraciones globales y acceso a datos.

  2. Nivel de entidad

    • Se activa por entidad específica.

    • Necesario para auditar cualquier registro de esa entidad.

    • Requiere que la auditoría de organización esté activada.

  3. Nivel de campo

    • Solo puede auditar campos si la auditoría de la organización y de la entidad está activada.

    • Permite rastrear cambios a nivel de valores individuales de campos concretos.

⚠️ En Dynamics 365 CRM, la auditoría funciona de forma jerárquica:
⚠️ Sin habilitar los niveles superiores, la auditoría de campos no funcionará, aunque se configure.

8.2 Funcionalidades de auditoría

  • Registro de actividades de usuarios: quién modificó qué y cuándo.

  • Seguimiento de transacciones y cambios críticos.

  • Base para control interno y cumplimiento de normativas.

8.3 Confusiones frecuentes en examen

  • Diferenciar los tres niveles de auditoría: organización vs entidad vs campo.

  • Comprender que auditoría de nivel de organización cubre configuración y sistema, no solo datos de registros.

  • Distinguir auditoría de entidad (registros completos) de auditoría de campo (valores individuales).

9. Incidencias, identificación y resolución

Las incidencias son problemas, errores o anomalías detectadas en el sistema ERP/CRM que pueden afectar la operación normal de la empresa.

9.1 Identificación

  • Detectar y registrar la incidencia.

  • Puede ser manual o mediante alertas automáticas del sistema.

9.2 Resolución

  • Aplicar acciones correctivas o ajustes necesarios.

  • Garantizar que la operación del sistema vuelva a la normalidad.

  • Mantener la integridad de los datos y la continuidad operativa.

Nota: Este tema tiene peso bajo en examen, por lo que basta con comprender los conceptos generales de incidencia, identificación y resolución.

10. Procesos de extracción de datos en sistemas ERP/CRM y automatización

Este bloque aborda cómo se preparan, organizan y automatizan los datos para su análisis y generación de informes financieros.

10.1 Plantilla de columnas

  • Define la estructura de los datos que se van a extraer.

  • Facilita la comparación de esquemas de cuentas.

  • No genera el informe directamente, solo sirve como formato o guía.

10.2 Configuración de un esquema de cuentas

  • Organiza el plan de cuentas y permite generar informes financieros.

  • Componentes principales:

    • Nombre del esquema

    • Líneas del esquema

    • Plantilla de columnas

10.3 Extracción y automatización

  • Los datos se extraen de tablas y entidades del sistema.

  • Plantillas y esquemas permiten automatizar la preparación de informes, evitando la extracción manual repetitiva.

  • Se pueden generar informes repetitivos o personalizados de manera más eficiente.

10.4 Confusiones frecuentes

  • Plantilla de columnas ≠ Report:

    • Plantilla → estructura para organizar y extraer datos

    • Report → resultado final imprimible/exportable

  • Comparación de esquemas → informa financieramente, no solo organiza cuentas.

  • Diferenciar extracción manual de datos de automatización mediante plantillas y esquemas.

11. Inteligencia de Negocio (Business Intelligence – BI)

La Inteligencia de Negocio (BI) es el conjunto de procesos, herramientas y metodologías que permiten transformar datos en información útil para la toma de decisiones empresariales.

11.1 Funcionalidad principal

  • Consolidar información de distintas fuentes (ERP, CRM, bases de datos).

  • Analizar patrones, tendencias y resultados históricos.

  • Visualizar la información mediante informes, gráficos y dashboards.

  • Facilitar la toma de decisiones basada en datos reales.

11.2 Relación con otros bloques

  • Se alimenta de esquemas de cuentas, informes y consultas.

  • Utiliza datos ya organizados y auditados.

  • Complementa la visualización de gráficos y dashboards.

Nota: Tema de contexto estratégico; no hay preguntas directas en los exámenes revisados, pero es importante comprender su propósito y función dentro del sistema.

  • Campos en Nav: BigInteger, BLOB, Boolean, Date, DateFormula, DateTime, Integer, Option, Text, Decimal

2. Tablas y vistas

  • Almacenan datos de clientes, productos, transacciones

  • Base de formularios, consultas e informes

3. Consultas de acceso a datos

  • Objetos: Queries/Consultas

  • Permiten extraer información de tablas

  • Base para informes, gráficos y BI

4. Interfaces y formularios

  • Visualizan y permiten editar datos: Pages/Formularios

  • Componentes: Encabezado, Cuerpo, Pie

  • Campos organizados en Secciones

  • Listados → característica principal de la interfaz

5. Informes y esquemas de cuentas

  • Report → generar PDF/Excel

  • Esquema de cuentas: organiza cuentas + genera informes

  • Componentes: Nombre, Líneas, Plantilla de columnas

  • Comparación de esquemas → informes financieros personalizados

  • Plantilla ≠ Report → solo estructura

6. Gestión operativa

  • Procesos diarios: Productos, Compras, Ajustes contables, Movimientos, Ventas/Clientes

  • Ficha de cliente → actualizada mediante transacción

7. Gráficos

  • Tipos: Sistema (predefinidos), Personales (usuario)

  • Reflejan datos de consultas e informes

  • Facilitan interpretación rápida y visual

8. Auditoría

  • Niveles:

    • Organización → sistema completo, debe activarse primero

    • Entidad → registros específicos, requiere organización activada

    • Campo → valores individuales, requiere organización + entidad activadas

  • Función: rastrear quién, qué y cuándo, control interno

  • Confusiones: diferenciar nivel y alcance de cada auditoría

9. Incidencias

  • Problemas o errores detectados en el sistema

  • Identificación → registro manual o automático

  • Resolución → acciones correctivas para restaurar operación

10. Extracción de datos y automatización

  • Plantilla de columnas → define estructura de datos

  • Esquemas de cuentas → organizan cuentas + generan informes

  • Automatiza informes repetitivos y personalizados

  • Confusión frecuente: plantilla ≠ Report

11. Business Intelligence (BI)

  • Consolida y analiza datos para tomar decisiones

  • Fuentes: ERP, CRM, tablas, consultas, informes

  • Salidas: informes, dashboards, gráficos

  • Concepto clave: analizar información histórica y tendencias
