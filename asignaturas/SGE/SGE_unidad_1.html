1. Conceptos básicos y definiciones

1.1. ERP (Enterprise Resource Planning)

  • Definición: Paquetes de sistemas de información configurables que integran información y procesos de negocio dentro y entre áreas funcionales de una organización.

  • Función principal: Gestionar procesos internos de la empresa (finanzas, contabilidad, compras, ventas, inventario, producción, RRHH ) en un único sistema.

  • Integralidad ⚠️: Un ERP es integral si centraliza todas las áreas funcionales, evitando duplicidades y garantizando consistencia de datos.

  • Configurabilidad ⚠️: Permite adaptar módulos y procesos a la empresa sin programar desde cero, reduciendo costes y tiempo de implementación.

  • Objetivos:

    • Optimizar procesos empresariales

    • Compartir información entre todas las áreas

    • Mantener integridad de datos y trazabilidad

1.2. CRM (Customer Relationship Management)

  • Definición: Sistemas destinados a gestionar relaciones y procesos comerciales con clientes, incluyendo captación, seguimiento y fidelización.

  • Función principal: Gestionar oportunidades, clientes, pipeline de ventas, segmentación y comunicaciones.

  • Visión 360° del cliente ⚠️: Acceso completo e integrado a toda la información del cliente (historial de compras, interacciones, preferencias).

  • Objetivos:

    • Incrementar ventas y fidelización

    • Mejorar la satisfacción del cliente

    • Integrar información para la toma de decisiones estratégicas

  • Implantación ⚠️: Menor que ERP debido a complejidad multidisciplinar y reticencias del equipo comercial.

1.3. Diferencias clave ERP vs CRM ⚠️

Concepto ERP CRM
Propósito Gestión interna de procesos y recursos Gestión de relaciones y oportunidades comerciales
Áreas cubiertas Finanzas, producción, inventario, RRHH, compras, ventas Ventas, marketing, atención al cliente, soporte
Nivel de implantación Mayor, procesos internos más maduros Menor, requiere cambios culturales y colaboración comercial
Usuarios clave Gerencia, operaciones, contabilidad Comerciales, marketing, servicio al cliente
Datos principales Inventario, órdenes, contabilidad Clientes, oportunidades, comunicaciones

  • ⚠️ Nemotecnia sugerida:

    • ERP → Empresa Rige Procesos

    • CRM → Clientes y Relaciones Mejoradas

1.4. Relación con preguntas tipo examen

  • “El ___ integra todas las funcionalidades para la gestión de procesos de negocio” → ERP

  • Funcionalidades de lead management → CRM

  • Implantación CRM menor que ERP → preguntas sobre adopción y reticencias

  • Visión 360° del cliente → concepto clave para preguntas de CRM

2. Tipos y ejemplos de sistemas ERP/CRM

2.1. Principales sistemas ERP/CRM en el mercado

  • ⚠️ Odoo

    • Tipo: ERP-CRM completo, modular

    • Arquitectura: MVC (Modelo-Vista-Controlador)

    • Lenguaje: Python

    • Base de datos: PostgreSQL

    • Licencia: LGPL v3 (Community)

    • Modularidad: Finanzas, compras, ventas, inventario, proyectos, CRM

    • ⚠️ Flujo integración: ventas/presupuesto → CRM, inventario → compras, contabilidad → registros financieros

  • Microsoft Dynamics 365

    • Componentes: Business Central (ERP) y Dynamics 365 CRM

    • Lenguaje extensiones: AL (C/AL descontinuado)

    • Base de datos: SQL Server ≥ 2008

    • Arquitectura: cliente-servidor y web

  • SugarCRM

    • Tipo: CRM

    • Base de datos: MySQL

    • Servidor web: Apache

2.2. Módulos principales y funciones

ERP

  • ⚠️ Finanzas / Contabilidad: general, analítica, presupuestaria

  • Compras y abastecimiento

  • Inventario y almacenes: control multiubicación, stock mínimo

  • Producción: planificación de necesidades (MRP)

  • RRHH y nóminas

CRM

  • Gestión de oportunidades (Lead Management) ⚠️

    • Registro y seguimiento de prospectos hasta el cierre

  • Segmentación de clientes

  • Gestión de campañas y comunicaciones

  • Visión 360° del cliente ⚠️

2.3. Flujos de integración entre módulos

  • Ventas / Presupuestos → CRM → Notificación a contabilidad e inventario

  • Compras → Inventario → Finanzas (contabilidad de cuentas por pagar)

  • Proyectos → Recursos asignados → CRM / ERP (seguimiento financiero y productivo)

⚠️ Diagrama de flujo conceptual (simplificado):

CRM (Clientes / Ventas / Oportunidades)
       │
       ▼
ERP (Inventario / Compras / Contabilidad)
       │
       ▼
Reportes / Dashboard / Scoreboards

  • Permite centralizar datos y procesos evitando duplicidades y errores.

2.4. Relación con preguntas tipo examen

  • “¿Cuál módulo se encuentra en ERP orientado a fabricación?” → MRP / Producción

  • “Funcionalidades de gestión de oportunidades” → CRM Lead Management

  • “Integración ventas/presupuestos en Odoo” → CRM ⚠️ (flujo principal)

  • Licencias Odoo → LGPL v3

  • Requisitos base de Dynamics 365 y SugarCRM → base de datos y servidor web

3. Arquitectura y capas de ERP/CRM

3.1. Tipos de arquitectura histórica y moderna

  • ⚠️ Cliente/Servidor:

    • Servidor centralizado que gestiona lógica de negocio y base de datos

    • Clientes instalados en los equipos de usuarios

    • Requiere conexión en red y compatibilidad de sistema operativo

    • Preguntas tipo examen: “Arquitectura ERP-CRM: Cliente/Servidor vs Monopuesto”

  • Monopuesto:

    • Aplicaciones instaladas en un único equipo, sin acceso multiusuario

    • Históricas, ya casi no se usan en ERP/CRM modernos

  • ⚠️ Nemotecnia:

    • Cliente/Servidor = centralización + multiusuario

    • Monopuesto = uso único, local

4. Integración de módulos y flujos en ERP/CRM

4.1. Principios de integración

  • ⚠️ Integración: intercambio de información entre módulos para evitar duplicidades y errores, garantizando consistencia de datos.

  • Los ERP/CRM modernos permiten automatizar procesos interdepartamentales (ventas, compras, contabilidad, inventario, proyectos).

  • Ejemplo clave: Odoo y Dynamics 365 Business Central, donde ventas, presupuesto e inventario se sincronizan automáticamente con contabilidad y compras.

4.2. Flujos de información típicos en Odoo

  • Ventas / Presupuestos → CRM:

    • Seguimiento de oportunidades y clientes

    • Generación automática de facturas y órdenes de venta

  • Inventario → Compras:

    • Stock mínimo genera órdenes automáticas a proveedores

    • Movimientos de productos registrados en contabilidad

  • Contabilidad → Finanzas:

    • Todos los movimientos de ventas, compras y proyectos se reflejan automáticamente

  • Proyectos → Recursos / CRM:

    • Asignación de tareas y seguimiento de costos

⚠️ Diagrama de flujo conceptual Odoo:

[CRM: Clientes / Ventas]
         │
         ▼
[ERP: Inventario / Compras / Contabilidad]
         │
         ▼
[Proyectos / Finanzas / Reportes]

4.3. Flujos de integración en Dynamics 365

  • Business Central (ERP) → Finanzas, inventario, compras, producción

  • Dynamics CRM → clientes, oportunidades, pipeline de ventas

  • Integración: los procesos de venta y presupuesto se reflejan en ERP para facturación, inventario y contabilidad

⚠️ Notas examen:

  • Preguntas sobre “con quién se integran los procesos de venta y presupuesto en Odoo” → CRM

  • Flujos de trabajo (workflow) permiten automatizar notificaciones y registros contables

  • Todas las integraciones deben garantizar trazabilidad y visibilidad 360°

4.4. Errores comunes en integración

  • No sincronizar correctamente ventas y contabilidad → duplicidad de datos

  • No configurar stock mínimo → errores en órdenes de compra automáticas

  • Flujo de clientes incompleto → pérdida de información de oportunidades

4.5. Relación con preguntas tipo examen

  • “Integración ventas/presupuestos en Odoo” → CRM

  • “Ventajas de implantar ERP” → centralización de información y automatización de flujos

  • “Lead management” → seguimiento de oportunidades en CRM integrado a ERP

  • Preguntas sobre workflows → reflejan flujo de procesos automáticos

5. Seguridad y permisos en ERP/CRM

5.1. Principios de seguridad

  • ⚠️ Seguridad: garantiza que solo usuarios autorizados accedan a datos y procesos del ERP/CRM.

  • Componentes principales:

    • Usuarios: cuentas individuales del sistema

    • Roles / Perfiles de seguridad: agrupan permisos y privilegios para asignación masiva

    • Políticas de sistema operativo / LDAP / Active Directory: integran autenticación y control de acceso externo

  • Objetivo: proteger información sensible, garantizar integridad y trazabilidad.

5.2. Seguridad en Dynamics 365 CRM

  • Niveles de seguridad:

    1. Usuarios del sistema ⚠️: cuentas internas de CRM

    2. Roles de seguridad: permisos agrupados para usuarios

    3. Política de seguridad de Windows / Active Directory / LDAP: autenticación y control de acceso externo

  • Preguntas de examen:

    1. “Seguridad basada en…” → opción correcta según temario: Todas las respuestas son correctas

    2. Roles de seguridad permiten asignar permisos de forma centralizada

5.3. Seguridad en Dynamics 365 Business Central

  • ⚠️ Rol de seguridad: agrupa permisos individuales para asignarlos a usuarios

  • Control de acceso sobre módulos ERP: finanzas, inventario, compras, ventas

  • Preinstalaciones y configuraciones previas necesarias:

    • Servidor SQL compatible

    • Sistema operativo compatible (Windows Server / Windows 7+)

    • Requisitos previos instalados (IIS, .NET, SSRS según versión)

5.4. Seguridad en Odoo

  • Usuarios con roles asignados a módulos (CRM, ventas, contabilidad)

  • Acceso a módulos según permisos definidos

  • Integración con autenticación externa opcional (LDAP/Active Directory)

5.5. Errores comunes en seguridad

  • ⚠️ Asignar permisos directamente a usuarios en lugar de roles → difícil mantenimiento

  • No configurar roles correctamente → exposición de datos sensibles

  • Ignorar autenticación externa → riesgo de acceso no autorizado

5.6. Relación con preguntas tipo examen

  • “Seguridad basada en…” → Dynamics 365 CRM → Todas las respuestas son correctas

  • “Elemento que agrupa permisos en Business Central” → Rol de seguridad

  • Preguntas sobre preinstalaciones y compatibilidad → preparar lista de requisitos mínimos de instalación

  • Licencia incorrecta → restricciones de módulos o número de usuarios

7. Conceptos transversales y examen

7.1. Visión 360° del cliente ⚠️

  • Definición: Acceso completo e integrado a toda la información del cliente, incluyendo historial de compras, interacciones, preferencias y comunicaciones.

  • Relacionado con CRM → permite mejorar la fidelización y la toma de decisiones comerciales.

  • Preguntas de examen típicas: “En un sistema CRM, ¿qué significa visión 360° del cliente?” → respuesta: acceso integral a toda la información del cliente.

7.2. Offshoring

  • Definición: Externalización de procesos o servicios a otro país para reducir costes o acceder a talento especializado.

  • Ejemplo: desarrollo de módulos ERP o soporte de CRM desde otro país.

  • Pregunta examen: “Término que describe externalización internacional” → Offshoring.

7.3. Scoreboards vs ERP

  • ⚠️ Diferencia clave:

    • Scoreboard: herramienta de control y visualización de indicadores

    • ERP: gestión integral de procesos y datos de negocio

  • Pregunta examen: “Diferencia entre ERP y Scoreboard” → ERP gestiona procesos, Scoreboard solo indicadores

7.4. Nivel de implantación CRM vs ERP ⚠️

  • CRM: menor implantación que ERP → depende de colaboración comercial y enfoque multidisciplinar

  • ERP: mayor implantación → procesos internos maduros y centralizados

  • Preguntas examen: “Nivel de implantación de CRM respecto a ERP” → CRM más reducido

7.5. Ventajas de implantar ERP/CRM

  • Centralización de información

  • Eliminación de duplicidades

  • Automatización de flujos de trabajo

  • Trazabilidad completa de procesos

  • Mejor toma de decisiones estratégicas y operativas

7.6. Errores comunes en conceptos transversales

  • ⚠️ Confundir ERP con CRM → recordar nemotecnia: ERP = Empresa Rige Procesos, CRM = Clientes y Relaciones Mejoradas

  • Confundir Scoreboard con ERP → Scoreboard solo indicadores

  • Ignorar offshoring como concepto de reducción de costes → pregunta frecuente de examen

7.7. Relación con preguntas tipo examen

  • “Visión 360° del cliente” → CRM

  • “Offshoring” → externalización internacional

  • “Nivel de implantación CRM vs ERP” → menor implantación CRM

  • “Diferencia ERP vs Scoreboard” → ERP gestiona procesos completos

1. Conceptos básicos

  • ⚠️ CRM: gestiona clientes, oportunidades, ventas, marketing.

  • Visión 360° cliente ⚠️: acceso completo a historial e interacciones.

  • ERP integral ⚠️: centraliza todas las áreas, evita duplicidades.

  • Configurabilidad ⚠️: adapta módulos sin programar desde cero.

  • Nemotecnia: ERP = Empresa Rige Procesos, CRM = Clientes y Relaciones Mejoradas.

2. Tipos y ejemplos de sistemas

  • Odoo: ERP-CRM, modular, MVC, Python, PostgreSQL, LGPL v3.

  • Dynamics 365: ERP (Business Central) y CRM; SQL Server ≥2008; AL; cliente-servidor/web.

  • SugarCRM: CRM puro, MySQL, Apache.

Módulos

  • ERP: Finanzas, Compras, Inventario, Producción, RRHH.

  • CRM: Lead Management, Segmentación, Pipeline, Visión 360° cliente.

Flujos

  • Ventas/Presupuesto → CRM → ERP → Contabilidad/Inventario.

  • Inventario mínimo → Compras automáticas → Finanzas.

  • Proyectos → Recursos → CRM/ERP.

3. Arquitectura

  • Cliente/Servidor ⚠️: servidor central + clientes; multiusuario.

  • Monopuesto: un solo equipo, histórico.

  • CRM: Capas → Aplicación Web → Servidor apps → Base de datos.

  • ERP: Capas → Sistema operativo → Modelo/Componentes → Base de datos.

  • Odoo MVC ⚠️: Modelo(PostgreSQL) → Controlador(Python) → Vista(XML).

4. Integración

  • Automatiza flujos entre módulos.

  • Evita duplicidad y errores.

  • ⚠️ Odoo: ventas/presupuesto → CRM; inventario → compras → contabilidad.

  • ⚠️ Dynamics 365: ventas → ERP → facturación, inventario, contabilidad.

  • Workflow = notificaciones y registros automáticos.

5. Seguridad y permisos

  • Dinámica 365 CRM: usuarios, roles, políticas Windows/LDAP/AD.

  • Business Central: roles de seguridad agrupan permisos.

  • Odoo: roles de módulo + autenticación opcional LDAP/AD.

  • ⚠️ Errores comunes: permisos directos a usuarios, roles mal configurados, ignorar autenticación externa.
