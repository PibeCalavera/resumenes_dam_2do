1. Conceptos fundamentales de instalación de software

1.1. Definición de instalador

Es un paquete software cuyo objetivo es guiar al usuario final en el proceso de implantación de una aplicación en su sistema operativo.

Incluye operaciones como:

  • Copia de archivos esenciales.

  • Configuración de la aplicación.

  • Creación de accesos directos y entradas de registro.

  • Posible interacción con el usuario mediante interfaz gráfica o instalación desatendida.

1.2. Tipos de instaladores

MSI (Windows Installer)

  • Instalador nativo de Windows.

  • Soporta instalación desatendida o guiada.

  • Permite instalación modular (selección de características y componentes).

  • Soporta rollback, reparación y actualización parcial.

  • Integrable en despliegues corporativos mediante políticas de grupo (GPO).

EXE

Ejecutables que pueden ser:

  • Bootstrapper: lanzador que prepara el entorno y llama al instalador principal.

  • Personalizado: lógica de instalación propia, más flexible pero menos estandarizada.

JAR (Java)

  • Contiene clases y recursos empaquetados de la aplicación Java.

  • Puede convertirse a EXE mediante herramientas como JSmooth para facilitar ejecución en Windows.

Inno Setup

  • Herramienta de generación de instaladores EXE.

  • Permite: creación de accesos directos, configuración de interfaz, desinstalación limpia y scripts adicionales.

1.3. Elementos de instalación

  • Accesos directos: facilitan el lanzamiento de la aplicación.

  • Interfaz de usuario: guía al usuario durante la instalación.

  • Ubicación de instalación: carpeta destino en disco.

  • Idioma de la interfaz del instalador: opcional según el instalador.

  • Licencia: aceptación obligatoria en la mayoría de casos.

2. Jerarquía y modularidad en Windows Installer

2.1. Jerarquía de instalación

Windows Installer organiza los elementos instalables siguiendo una estructura jerárquica:

Producto software

  • Nivel superior.

  • Representa la aplicación completa que se va a instalar.

  • Contiene todas las características y componentes.

Característica (Feature)

  • Unidad lógica y funcional.

  • Visible y seleccionable por el usuario durante la instalación.

  • Agrupa uno o varios componentes.

  • Permite al usuario personalizar la instalación activando o desactivando ciertas funcionalidades.

  • Ejemplo: en un editor de texto, la característica “Diccionarios” puede ser opcional.

Componente

  • Unidad mínima técnica instalable.

  • No necesariamente visible para el usuario.

  • Incluye archivos, claves de registro y recursos necesarios para una funcionalidad específica.

  • Garantiza que la instalación, reparación y actualización parcial se realicen correctamente.

  • Ejemplo: un archivo DLL necesario para la función de impresión dentro de la característica “Impresión”.

2.2. Flexibilidad que aporta la modularidad

  • Instalación selectiva de características según necesidades del usuario o del entorno.

  • Instalación bajo demanda de componentes, sin reinstalar todo el producto.

  • Actualizaciones parciales: solo se reemplazan componentes afectados por la nueva versión.

  • Reparación: si algún componente se corrompe, puede reinstalarse sin afectar otras partes del producto.

2.3. Diferencias principales

  • Característica → perspectiva del usuario, seleccionable y visible.

  • Componente → perspectiva técnica, unidad mínima de instalación y mantenimiento.

  • Confusión frecuente: recordar que no se debe confundir la visibilidad con la mínima unidad técnica.

3. Instaladores MSI frente a EXE

3.1. MSI (Windows Installer)

Instalador nativo de Windows.

Ventajas principales en entornos corporativos:

  • Instalación modular y personalizada: permite seleccionar características y componentes específicos.

  • Desinstalación limpia y rollback automático: garantiza que no queden restos del software.

  • Integración con despliegues centralizados mediante políticas de grupo (GPO).

  • Actualizaciones parciales: solo se reemplazan los componentes necesarios.

Características destacadas:

  • Genera puntos de restauración automáticos.

  • Compatible con instalación desatendida para despliegues masivos.

  • Permite reparación de componentes dañados sin reinstalar todo el producto.

3.2. EXE (ejecutable)

Más flexible y simple para usuarios finales. Puede ser:

Bootstrapper:

Lanzador que prepara el entorno y descarga/instala MSI u otros paquetes.

  • Beneficio: ocupa menos espacio y centraliza la ejecución en un único archivo.

Personalizado:

Contiene toda la lógica de instalación en un ejecutable.

  • Limitaciones frente a MSI:

    • No siempre garantiza rollback automático.

    • Puede no integrarse en políticas corporativas ni permitir instalación desatendida completa.

3.3. Comparación resumida

Característica MSI EXE
Modularidad Alta (Producto → Feature → Componente) Depende del programador
Instalación desatendida Sí Solo si se programa
Desinstalación limpia Sí No siempre
Integración corporativa Sí (GPO, Active Directory) Limitada
Simplicidad para usuario final Media Alta
Tamaño de archivo Variable Generalmente mayor si contiene todo el instalador

3.4. Conclusión práctica

  • MSI: ideal para entornos empresariales y control centralizado.

  • EXE: más adecuado para usuarios individuales, descargas simples o bootstrapper.

4. Distribución de aplicaciones Java y herramientas asociadas

4.1. Empaquetado de aplicaciones Java

  • Una aplicación Java se compila en archivos .class y se empaqueta en un archivo .jar (Java ARchive) para su distribución.

  • Ventajas del JAR:

    • Contiene todas las clases y recursos necesarios.

    • Permite ejecución en cualquier máquina con JVM compatible.

  • NetBeans:

    • Carpeta predeterminada para el JAR empaquetado: dist/.

    • En proyectos Maven, se genera en target/.

4.2. Transformación a ejecutable Windows (EXE)

  • Herramientas como JSmooth permiten crear un launcher EXE a partir del JAR.

    • Define la clase principal de la aplicación.

    • Facilita la ejecución directa en Windows sin que el usuario final necesite invocar la JVM manualmente.

  • Beneficio principal: un único archivo ejecutable, más cómodo para el usuario.

4.3. Creación de instaladores EXE para Windows

  • Inno Setup: herramienta para generar instaladores EXE.

    • Soporta:

      • Creación de accesos directos.

      • Interfaz gráfica durante la instalación.

      • Desinstalación limpia.

      • Copia de archivos y configuración de registro.

  • Permite distribuir aplicaciones Java en Windows de manera profesional y segura.

4.4. Opciones de distribución de aplicaciones Java

  • Servidor web: descarga directa, control de versiones y seguridad.

  • Correo electrónico: solo recomendable para pocos usuarios, no escalable.

  • Tiendas de aplicaciones (Google Play, Microsoft Store, Apple App Store):

    • Proporcionan firma verificada, actualizaciones automáticas y distribución segura.

  • Medios físicos: USB o CD/DVD (menos habitual hoy, usado en entornos aislados).

5. Seguridad y firmas digitales

5.1. Objetivo de la firma digital

  • Garantizar integridad, autenticidad y confianza en los archivos instalables.

  • Permite al usuario verificar que:

    • El archivo no ha sido modificado desde su creación.

    • Proviene del autor legítimo.

5.2. Conceptos clave

  • Par de claves asimétrico:

    • Clave privada: utilizada por el autor para firmar el archivo.

    • Clave pública: incluida en el archivo (JAR) para verificación por la JVM o el instalador.

    • La operación de firma y verificación solo funciona combinando estas dos claves de manera complementaria.

5.3. Certificado digital:

  • Credencial firmada por una Autoridad de Certificación (CA).

  • Indica quién es el propietario de la clave pública y valida la identidad del autor.

5.4. Beneficios prácticos para el usuario final

  • Evita la ejecución de archivos modificados o maliciosos.

  • Permite confiar en la fuente oficial del software.

  • Mejora la seguridad en entornos corporativos y tiendas de aplicaciones.

5.5. Implementación en aplicaciones Java

  • La llave pública y el certificado se incluyen en el JAR.

  • La llave privada nunca se distribuye, solo el autor la posee.

  • La JVM o el instalador verifica la firma digital antes de ejecutar la aplicación.

6. Flexibilidad y personalización de la instalación

6.1. Opciones durante la instalación

  • Selección de características: el usuario puede elegir qué funcionalidades instalar.

  • Ubicación de instalación: carpeta de destino en disco, a menudo modificable.

  • Idioma de la interfaz del instalador: opcional según el instalador.

  • Creación de accesos directos: escritorio, menú inicio; generalmente opcional.

  • Aceptación de licencia: normalmente obligatoria.

6.2. Beneficios de la modularidad

  • Instalación selectiva: solo se instalan las funciones requeridas, optimizando espacio y tiempo.

  • Instalación bajo demanda: componentes adicionales pueden añadirse después sin reinstalar todo el producto.

  • Actualizaciones parciales: solo los componentes modificados se actualizan.

  • Reparación: un componente dañado puede reinstalarse sin afectar al resto de la aplicación.

6.3. Diferencia entre EXE y MSI en flexibilidad

  • MSI: control granular, soporte para rollback, instalación corporativa, actualización y reparación parcial.

  • EXE: más simple, menos integrado, útil para instalaciones individuales o bootstrapper, pero menos control de mantenimiento.

6.4. Consideraciones de usabilidad y seguridad

  • Opciones como creación de accesos directos o idioma son configurables, no críticas para la instalación.

  • Opciones obligatorias (licencia, ubicación, compatibilidad del sistema operativo) garantizan que la aplicación funcione correctamente.

  • Sistemas corporativos o servidores pueden restringir ejecución de binarios no verificados, incluso si el instalador es EXE.

7. Distribución y métodos de entrega de aplicaciones

7.1. Métodos tradicionales de distribución

  • Servidor web: la forma más común y escalable.

    • Ventajas: control de versiones, seguridad, facilidad de descarga.

  • Correo electrónico: solo recomendable para pocos usuarios, por limitaciones de tamaño, seguridad y escalabilidad.

  • Medios físicos: USB, CD/DVD.

    • Usado en entornos aislados o sin acceso a internet.

7.2. Distribución moderna

  • Tiendas de aplicaciones: Google Play, Microsoft Store, Apple App Store.

    • Beneficios:

      • Instalación segura con firma verificada.

      • Actualizaciones automáticas.

      • Control de versiones y compatibilidad garantizada.

      • Confianza para el usuario final al reducir riesgos de malware.

7.3. Formatos recomendados para distribución segura

  • MSI: para entornos corporativos, instalación gestionada, soporte de rollback y actualizaciones.

  • EXE: conveniente para descargas simples o bootstrapper que descarga MSI u otros instaladores.

  • JAR: para aplicaciones Java; puede ser firmado digitalmente para seguridad.

7.4. Consideraciones finales para la distribución

  • Elegir el método según número de usuarios, seguridad y facilidad de actualización.

  • Los sistemas seguros (servidores corporativos) pueden bloquear ejecutables no verificados, por lo que la firma digital y los instaladores estándar son esenciales.

  • La correcta combinación de formato de instalador y método de distribución asegura compatibilidad, facilidad de uso y seguridad para el usuario final.
