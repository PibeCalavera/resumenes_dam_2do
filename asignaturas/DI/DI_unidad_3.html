1. Principios de diseño de interfaces

Los principios de diseño son guías fundamentales para crear interfaces gráficas eficientes, comprensibles y accesibles. Se aplican a todos los elementos de la interfaz, desde botones hasta menús y diálogos.

1.1 Correspondencia (Mapping)

  • Define que la acción de un control debe reflejar intuitivamente el efecto que produce en el sistema.

  • Ejemplo: presionar un botón hacia abajo activa la acción; mover un deslizador a la derecha aumenta un valor.

  • Beneficio: predicción intuitiva, reducción de errores y aprendizaje más rápido del usuario.

1.2 Retroalimentación (Feedback)

  • El sistema debe informar al usuario sobre el resultado de sus acciones, evitando incertidumbre.

  • Ejemplos: mensaje de éxito, indicador de progreso, cambio visual de un control.

  • Beneficio: asegura que el usuario sepa que la acción fue realizada correctamente o necesita corrección.

1.3 Consistencia y Previsibilidad

  • Los elementos de la interfaz deben mantener comportamiento y estilo uniforme, dentro de la aplicación y entre aplicaciones similares.

  • Ejemplo: menús siempre en la parte superior, atajos de teclado comunes como CTRL+S para guardar.

  • Beneficio: reduce la curva de aprendizaje y permite la transferencia de conocimientos entre aplicaciones.

1.4 Eficiencia

  • Facilitar la ejecución rápida de tareas frecuentes, especialmente para usuarios avanzados.

  • Herramientas clave:

    • Atajos de teclado (shortcuts): ejecutar acciones sin usar el ratón.

    • Barras de herramientas (toolbars): iconos de acceso rápido a funciones frecuentes.

  • Beneficio: aumenta productividad y fluidez en el uso de la aplicación.

1.5 Accesibilidad

  • La interfaz debe ser usable por todos los usuarios, incluyendo aquellos con limitaciones visuales, motoras o cognitivas.

  • Principios:

    • Texto legible y contrastado.

    • Controles fácilmente reconocibles y seleccionables.

    • Compatibilidad con herramientas de asistencia y estándares WCAG.

2. Controles y Componentes de UI

Los controles de interfaz permiten al usuario interactuar con la aplicación, ya sea ejecutando acciones, seleccionando valores o navegando por el contenido. Cada control tiene un propósito específico y se debe elegir según la tarea a realizar.

2.1 Botones y Botones de Comando

  • Función: Ejecutar una acción inmediata al ser presionados.

  • Uso correcto:

    • Texto conciso, generalmente en infinitivo en español: “Guardar”, “Cancelar”, “Enviar”.

    • Se puede asociar atajo de teclado para acciones frecuentes.

    • Tamaño y estilo consistente dentro de la aplicación.

  • Ejemplo: Botón “Guardar” que almacena los cambios en un documento.

2.2 Botones tipo radio (Radio Buttons)

  • Función: Permitir la selección de un único valor entre varias opciones mutuamente excluyentes.

  • Uso correcto:

    • Visualizar todas las opciones disponibles de un vistazo.

    • Adecuado para 2–7 opciones; si hay 8 o más, se recomienda una lista desplegable.

  • Ejemplo: Selección del método de pago: “Tarjeta”, “Paypal”, “Transferencia”.

2.3 Casillas de verificación (Checkboxes)

  • Función: Permitir la selección múltiple dentro de un conjunto de opciones.

  • Uso correcto:

    • Cada opción es independiente; el usuario puede marcar o desmarcar libremente.

    • Se deben usar cuando las opciones no son mutuamente excluyentes.

  • Ejemplo: Selección de intereses en un formulario: “Deportes”, “Música”, “Cine”.

2.4 Listas desplegables (Dropdown / Combo Box)

  • Función: Permitir seleccionar un único valor de un conjunto amplio de opciones.

  • Uso correcto:

    • Reducir saturación visual cuando el número de opciones es grande (8 o más).

    • El valor seleccionado queda visible cuando la lista está cerrada.

  • Ejemplo: Selección del país en un formulario de registro.

2.5 Campos de texto y etiquetas

  • Campos de texto (TextField / TextArea):

    • Permiten entrada editable de datos por parte del usuario.

    • Pueden configurarse como solo lectura para mostrar información no editable pero seleccionable y copiables.

  • Etiquetas (Label):

    • Mostrar texto informativo o descriptivo sin permitir edición.

2.6 Barras de herramientas (Toolbars)

  • Función: Proporcionar acceso rápido a funciones frecuentes mediante iconos.

  • Uso correcto:

    • Iconos reconocibles y coherentes con las funciones.

    • Complementa la barra de menú, pero no reemplaza la navegación completa.

  • Ejemplo: Barra con iconos “Guardar”, “Abrir”, “Imprimir” en un editor de texto.

2.7 Pestañas (Tabs)

  • Función: Organizar contenido en secciones dentro de la misma ventana, permitiendo navegar entre ellas sin abrir nuevas ventanas.

  • Uso correcto:

    • Colocar elementos comunes fuera de las pestañas.

    • Colocar elementos específicos de la sección dentro de la pestaña.

    • Evitar que el usuario pueda crear o borrar pestañas indiscriminadamente.

  • Ejemplo: Configuración de un software con pestañas “General”, “Seguridad”, “Avanzado”.

2.8 Cuadros de diálogo (Dialogs)

  • Función: Comunicar información o solicitar interacción específica al usuario.

  • Tipos:

    • Información: Mensajes informativos con icono de información.

    • Confirmación: Solicita decisión obligatoria, bloqueando la aplicación hasta la respuesta.

    • Advertencia / Error: Alertan de problemas o acciones críticas.

    • Excepción: Muestra errores del sistema de manera controlada.

  • Uso correcto:

    • Ser simples y enfocados en una única tarea.

    • Modales cuando la decisión es crítica.

  • Ejemplo: “¿Desea guardar los cambios antes de salir?”

3. Menús y Barras

Los menús y barras permiten acceder a funcionalidades y comandos de la aplicación de forma estructurada y rápida. Son elementos esenciales en aplicaciones de escritorio y en interfaces web complejas.

3.1 Barra de Menú (Menu Bar)

  • Ubicación: Parte superior de la ventana, siguiendo las convenciones de escritorio.

  • Función: Contener elementos jerárquicos (menús desplegables) con todas las opciones principales de la aplicación.

  • Buenas prácticas:

    • Nombres claros, concisos y reconocibles.

    • Evitar componentes complejos dentro de los menús.

    • Mantener consistencia con otras aplicaciones para reducir la curva de aprendizaje.

  • Ejemplo: Menú principal con “Archivo”, “Edición”, “Ver”, “Ayuda”.

3.2 Menú Ribbon / Cinta

  • Ubicación y estructura: Basado en pestañas que agrupan comandos por tareas o funcionalidades.

  • Ventaja: Reduce la profundidad de navegación en comparación con menús tradicionales, mostrando grupos de herramientas relacionados en una sola vista.

  • Ejemplo: Microsoft Office: pestañas “Inicio”, “Insertar”, “Revisar”, cada una con grupos de botones y comandos.

3.3 Barras de Herramientas (Toolbars)

  • Función: Acceso rápido a funciones frecuentes mediante iconos.

  • Complemento de la barra de menú: No reemplaza la navegación completa; facilita acciones repetitivas.

  • Ejemplo: Iconos “Guardar”, “Imprimir”, “Deshacer”, ubicados debajo de la barra de menú.

3.4 Atajos de Teclado (Shortcuts)

  • Función: Permiten ejecutar acciones frecuentes sin usar el ratón, aumentando la eficiencia.

  • Ejemplos comunes:

    • CTRL+S → Guardar

    • CTRL+C → Copiar

    • CTRL+V → Pegar

  • Buenas prácticas:

    • Atajos consistentes con estándares de escritorio.

    • Asociar teclas de acceso rápido visibles en los menús.

3.5 Principios y buenas prácticas

  • Consistencia: Mantener ubicación y comportamiento uniforme.

  • Claridad: Nombres de menús y elementos comprensibles.

  • Previsibilidad: Las acciones del menú deben reflejar la función que ejecutan.

  • Accesibilidad: Todos los elementos deben ser operables mediante teclado y con suficiente contraste visual.

3.6 Diferencias entre barra de menú y barra de herramientas

Elemento Función principal Forma de interacción Uso típico
Menu Bar Contiene todas las opciones de la aplicación Texto y submenús Organización jerárquica y navegación completa
Toolbar Acceso rápido a acciones frecuentes Iconos clicables Ejecución rápida de funciones repetitivas

4. Estilo y Presentación

El estilo de los componentes de interfaz determina cómo se percibe la aplicación visualmente y afecta directamente a la legibilidad, la usabilidad y la accesibilidad.

4.1 Propiedades de texto en JavaFX

  • Tamaño de fuente:

    • Propiedad CSS: -fx-font-size

    • Ejemplo: -fx-font-size: 14px;

    • Permite ajustar el tamaño para mejorar lectura y jerarquía visual.

  • Familia tipográfica:

    • Propiedad CSS: -fx-font-family

    • Ejemplo: -fx-font-family: "Arial";

    • Determina la apariencia y consistencia tipográfica de los controles.

  • Color de texto:

    • Propiedad CSS: -fx-text-fill

    • Formatos válidos:

      • Hexadecimal: #FF0000

      • Nombre de color: red, blue, white

      • RGB: rgb(255,0,0)

    • Permite asegurar contraste y visibilidad adecuada.

4.2 Contraste y accesibilidad

  • Es fundamental que el texto y los controles sean legibles frente al fondo.

  • Se recomienda evaluar contraste con herramientas como Colour Contrast Analyser 2.0.

  • Estándares de accesibilidad: WCAG (Web Content Accessibility Guidelines).

  • Beneficio: mejora la usabilidad para personas con dificultades visuales y asegura cumplimiento de normas internacionales.

4.3 Aplicación de estilo

  • CSS en JavaFX permite separar estilo y funcionalidad, mejorando mantenimiento y consistencia.

  • Ejemplo de estilo de botón:

.button {
    -fx-font-size: 14px;
    -fx-font-family: "Verdana";
    -fx-text-fill: #FFFFFF;
    -fx-background-color: #0078D7;
}

  • Ventaja: cambios de estilo globales sin modificar la lógica de la aplicación.

4.4 Principios de presentación

  • Legibilidad: texto visible y contrastado.

  • Consistencia: misma tipografía, tamaño y colores para tipos similares de información.

  • Claridad: uso de colores y fuentes que refuercen la jerarquía de información.

  • Neutralidad: evitar colores agresivos o llamativos innecesarios que distraigan al usuario.

5. Metáforas y objetos reconocibles

Las metáforas de interfaz son elementos visuales o conceptuales que permiten al usuario entender rápidamente la función de un control o área basándose en su experiencia previa con objetos del mundo real o con otras aplicaciones.

5.1 Concepto de metáfora en UI

  • Una metáfora relaciona una acción digital con un objeto o concepto conocido.

  • Beneficio: reduce la curva de aprendizaje, aumenta la intuición y la previsibilidad de la interfaz.

  • Ejemplo clásico: “Papelera de reciclaje” para eliminar archivos.

5.2 Tipos de metáforas comunes

  1. Metáforas de escritorio:

    • Carpetas → organización de archivos.

    • Archivos → documentos guardados.

    • Escritorio → área principal de trabajo.

  2. Metáforas de control:

    • Botones que parecen pulsables.

    • Deslizadores que emulan palancas o perillas.

  3. Iconos reconocibles:

    • Disquete → guardar (aunque menos usado hoy).

    • Lupa → búsqueda.

    • Sobre → correo o mensaje.

5.3 Buenas prácticas de uso

  • Usar objetos ya conocidos por los usuarios y mantener consistencia con otras aplicaciones.

  • Evitar metáforas confusas o demasiado abstractas que requieran explicación.

  • Acompañar iconos con texto cuando sea necesario para reforzar comprensión.

5.4 Relación con otros principios de UI

  • Consistencia: los iconos y metáforas deben funcionar igual en toda la aplicación.

  • Correspondencia (mapping): el efecto del control debe coincidir con la metáfora visual.

  • Accesibilidad: metáforas deben ser comprensibles incluso con ayudas visuales o lectores de pantalla.

6. Interacción y Usabilidad

La interacción y la usabilidad se refieren a cómo el usuario percibe, comprende y maneja la interfaz, y cómo la interfaz guía al usuario para realizar tareas de forma eficiente y segura.

6.1 Principios de interacción

  • Retroalimentación:

    • La interfaz debe informar siempre al usuario del resultado de sus acciones.

    • Ejemplo: indicador de progreso, mensaje de confirmación o cambio visual de un botón.

  • Correspondencia / Mapping:

    • La acción del control debe reflejar intuitivamente su efecto.

    • Ejemplo: presionar un botón hacia abajo ejecuta la acción indicada.

  • Consistencia y previsibilidad:

    • Elementos con misma función deben comportarse igual.

    • Ejemplo: CTRL+S siempre guarda, barras de herramientas y menús usan iconos estándar.

6.2 Agrupación y navegación

  • Agrupación lógica de controles:

    • Relacionar controles y elementos según su función para mejorar comprensión y velocidad.

    • Ejemplo: pestañas para secciones relacionadas, menús agrupando funciones similares.

  • Navegación clara:

    • Evitar que el usuario se pierda entre múltiples ventanas o paneles.

    • Ejemplo: pestañas, barras de herramientas y menús organizados jerárquicamente.

6.3 Diálogos y decisiones críticas

  • Cuadros de diálogo modales:

    • Bloquean la aplicación hasta que el usuario tome una decisión.

    • Se usan para acciones críticas que no pueden ser ignoradas (guardar cambios, confirmar eliminación).

  • Mensajes de alerta y advertencia:

    • Informativos → icono de información.

    • Advertencia → icono de advertencia.

    • Error → icono de error.

  • Garantiza control del flujo de la aplicación y seguridad de los datos.

6.4 Eficiencia y accesibilidad

  • Atajos de teclado: facilitan ejecución rápida de tareas frecuentes.

  • Toolbars: acceso inmediato a funciones recurrentes mediante iconos reconocibles.

  • Contraste y legibilidad: colores, tamaños y fuentes que permitan uso cómodo por todos los usuarios.

  • Compatibilidad con ayudas técnicas: lectores de pantalla, teclados alternativos, etc.

6.5 Buenas prácticas generales

  • Interfaz clara, consistente y predecible.

  • Mensajes y controles simples y directos.

  • Uso de metáforas y elementos reconocibles.

  • Minimizar sobrecarga visual y cognitiva.

  • Mantener jerarquía de información y flujo lógico de tareas.
